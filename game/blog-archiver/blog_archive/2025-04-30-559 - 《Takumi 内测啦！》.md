1、**Takumi 是啥？**

Takumi 是一个 Cli 形式 Code Agent，类似社区的 Claude Code、Codex、Aider 等，刚开始写他的时候还没那么多竞品的，现在越来越多了。Takumi 在功能层会逐渐和社区方案对齐，同时集成更多内部的平台、服务、框架和领域知识，满足内部场景的需要。希望他在 Cursor 之外也能成为大家的一个选择。

Takumi 和之前做 Bigfish 和 Umi 的思路类似，也是开源 + 内源的思路。我们对外的产品叫 takumi（暂定，同时目前还是私库开发，未公开），然后内部有一个 NeoCoder Cli，NeoCoder Cli = takumi + takumi-plugin-antgroup 。

Takumi 不少代码是 Takumi 自己写的。

2、**Takumi 有啥特性？**

基础功能。

*   和大模型交互，支持主流的 LLMs 和 Provider
*   内置 Code Agent 应该有的常用 Tools，含文件操作、代码搜索、命令执行等
*   支持 MCP（Auth 还不支持）
*   支持 Rule 文件，基于 `TAKUMI.md`
*   支持 Session 日志，用于排查问题和记录

进阶功能。

*   支持计划模式，参数是 `--plan`，执行复杂任务之前建议先确认计划
*   支持插件体系，比如你想增加上下文、修改配置、新增子命令等
*   支持 model 别名，当你要用外网的模型时会比较好用，比如 `takumi -m gemini`
*   `takumi watch` 会进入 watch 模式，在编辑器里写带 `AI!` 的注释即指令（参考自 Aider）
*   `takumi init` 会理解项目并生成简要的 `TAKUMI.md`，后续的 AI 操作会 follow 这个
*   `takumi ask` 如果你只想问问题而不想要修改代码
*   `takumi test` 和 `takumi lint` 会跑对应命令并修复错误
*   `takumi commit` 会做 git 提交（个人使用频率最高，因为他比我写的好）

蚂蚁特有功能。

*   内置 DeepSeek V3 和 QwQ-32B，分别用于大小任务，无需 `-m` 参数即可使用
*   内置 [bigfish-mcp](https://code.alipay.com/bigfish/bigfish-mcp)，含 20+ Tools，针对 Bigfish 4 项目自动开启（其他 fish 的后续 也会跟上）
*   内置雨燕监控

3、**如何快速上手？**

先安装（应该是极快的，仅几 M，且无依赖）。

```bash
$ npm install -g takumi
```

然后。

```bash
$ takumi
```

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/image-2025-04@main/uPic/c3.gif)

DONE 。

4、**个人经验。**

附一些个人经验，方便大家更好地上手 Takumi。

*   `takumi commit --stage --commit` 是个人使用频率最高的一个子命令，因为他写的 commit 信息比我好，原因之一是我们再 prompt 里限制了产出的字符长度，所以比较简洁，符合个人预期。
*   `takumi --quiet` 会进入非交互模式，执行完成即退出。
*   `takumi --plan` 还没那么好用，尚在打磨中。
*   `takumi --mcp <mcp_cmd_or_sse_url>` 可用于附加 mcp server 。
*   `takumi --codebase` 会用 repomix 把整个项目打包后问 LLMs，尤其适用于小项目 + 外部大 context 的 gemini 模型的组合。
*   现在的版本只有「自动 Approve」的模式，没有让用户做确认，后面会支持不同的 Approve 模式。所以现在用的时候大家需要稍微小心一点，比如用 git 保存好相关的代码。
*   如果要用外网的模型，在保证数据安全的基础上，推荐用免费的 gemini-2.5-pro-exp-03-25（多申请几个 key，换着用）、充 $5 送 $150/M 的 grok-3-fast-beta 。用 Claude 3.x Sonnet 现阶段会比较费钱，因为还没有做 Prompt Cache 的支持。
*   takumi 有记录 session log，方便自行排查问题或者找开发者定位问题。

5、**反馈和参与。**

目前 Takumi 还是一个测试的版本，有太多功能和 bug 想做和修复，但是时间和精力有限。大家在使用的过程中如果遇到问题，或者希望关注 Takumi 的最新进展，欢迎加我们的群反馈和交流。
