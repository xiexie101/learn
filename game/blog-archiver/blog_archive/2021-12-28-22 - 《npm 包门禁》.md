# 22 - 《npm 包门禁》

说下背景。这是下午聊到的不成熟的新想法，解「npm 包乱用、轮子乱造」的问题。就中台项目而言，前端在用的 npm 依赖差不多 300+，这 300+ 中质量参差不齐，同一个子领域中也有多个选择。同时开发者不清楚哪些好，哪些不好，哪些能用，哪些不能用。以及还有因为不知道用啥而造的低级轮子。

比如 lodash 不应该用，要改成 lodash-es；比如 @monaco-editor/react、react-monaco-editor、react-cdn-monaco-editor 之间应该推荐第一个，禁用第三个。

这些 npm 包需要进行归类、确定标准、整理质量公式、打分，通过分数评估质量。然后通过服务接入框架、工具、LOW CODE 平台等。

质量差库禁用🚫；同类库中质量不够的给警告⚠️，并推荐质量好的；使用质量好的库给👍🏻。长此以往，可以让内部生态更健康。
