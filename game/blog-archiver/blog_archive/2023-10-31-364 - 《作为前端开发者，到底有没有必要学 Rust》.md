# 364 - 《作为前端开发者，到底有没有必要学 Rust》

> 看到三元写的文章 [https://sourl.cn/fCTnYz](https://sourl.cn/fCTnYz) 「作为前端开发者，你没有必要学 Rust」后有些不同看法，做下记录。

1、例行先问了下 ChatGPT，答案是「作为一个前端开发者，学习 Rust 可能不是必需的，但绝对有其优点。根据你的个人和职业目标，这门语言可能会成为你技能库中一个有价值的补充。」真是会说话啊。

2、Rust 会带来更多机会。大部分同学确实不用学，但为啥你是属于大部分的那一部分？从前端社区的发展来看，大部分工具，能用 Rust 重写的都会用 Rust 重写。包括且不限于构建、压缩、转义、Lint、Prettier、甚至框架本身。届时，对于前端同学来说，不懂 Rust 肯定是能用，但也只限于能用。一旦你遇到 bug 或者需要去扩展他，或者需要基于他去做封装，或者你的业务有不一样的需求时你需要基于他做二次开发，这时你就只能求助会 Rust 的同学。

3、Rust 也会带来不同的视野。一个问题有很多解，如果只会 JavaScript，那就只能得出 JavaScript 的解。比如开发桌面 App，会 Rust 的同学会多一个 Tauri 的选择；比如开发 SWC 扩展，会 Rust 的同学会多一个 WASM 的选择；同时由于 Rust 的高性能，也会覆盖更多的可能性，比如之前我们会小心翼翼的不让 Babel 编译 node\_modules 下的文件，因为很慢，并且由此引发了一系列连带问题，包括补丁方案、压缩语法兼容等，换 Rust 后脑洞可能一下子就开了。

4、不是所有工具都会支持 JavaScript 的扩展。同时是否支持 JavaScript 扩展也并不能决定一个工具的好坏。不支持的原因很多，可能是开发成本，也可能是 by design，因为一旦上了 JavaScript 就会很慢。比如之前有人和我说某 Rust 构建工具比 Webpack 还慢，我问为啥，他说在 Rust 构建工具的基础上又增加了 Babel 满足业务需求，哈哈。

5、Rust 真的难么？比 JavaScript 难，但也还好吧。学过就忘的应该大部分是没有用起来吧。Rust 本身也在朝着通过编译器简化编程概念的方向走，未来只会更简单。同时现在有 ChatGPT 和 Copilot 等辅助工具，会问就等于会写了。另外，你也可以先学 Rust 里的 Easy Part，无脑 clone()，无需生命周期就已经能写很多东西和 Tauri 的接口层了。

个人观点，仅供参考。
