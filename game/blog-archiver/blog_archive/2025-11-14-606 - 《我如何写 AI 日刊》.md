# 606 - 《我如何写 AI 日刊》

维护 [modelwatch.dev](https://modelwatch.dev/) 一晃也好多个月了。出发点是因为日常就有跟进 ai 最新资讯的需求，然后顺便把收集到的信息分享出来让更多的人受益。而之所以能一直坚持写，也是因为这个过程很快，通常 10 分钟就能搞定。

![](https://pic.sorrycc.com/proxy/1763087881469-80325032.png)

背后是一套打磨过的 workflow，基于 ai。每天来了跑下 npm run generate，然后等着 ai 从 200+ 到 500 条 twitter 中整理出 30+ 资讯，我人工选 6-10 条，然后按回车发布。

![](https://pic.sorrycc.com/proxy/1763078810725-675134105.png)

说下怎么实现的。

1）建 twitter list 。ai 相关的资讯，twitter 应该是最快最新的。以下是我建的，目前收集了 78 位同学，会持续增加。

[https://x.com/i/lists/1602502639287435265](https://x.com/i/lists/1602502639287435265)

2）收集资讯。我是通过 [twitterapi.io](http://twitterapi.io) 收集 twitter list 的增量信息，用 api 的方式，这个服务还挺稳的，而且不贵，推荐。我设了上限 500 条，太多了没必要而且会触及 ai 的 context limit。

3）ai 筛选和整理。组织了下 prompt，然后让 ai 按我的风格和喜好整理资讯。这里用了 gemini 2.5 pro，因为 context limit 比较高，而且文字能力相对比较好。

4）格式化。因为我要发好几个渠道，telegram 和 modelwatch.dev ，前者是特殊的 markdown 标记，后者是 html，所以会分别格式化出几份来。

5）发布。telegram 是自动的， modelwatch.dev 目前是手动的。

Done。补一个完整 workflow 的[图示](https://pic.sorrycc.com/proxy/1762911210500-485632201.png)。
