6、Windsurf [/workflows](https://docs.windsurf.com/windsurf/cascade/workflows) 功能使用技巧分享。

```
Our devs have been getting a ton of value from using our /workflows feature since we launched it last month! Here are some of our favorite use cases and tips. ↓

---

Example workflow: /format-code Automatically run code formatters (like Prettier, Black) and linters (like ESLint, Flake8) before committing to maintain code style and catch errors early.

Example workflow: /manage-dependencies Automate the installation or updating of project dependencies based on a configuration file (e.g., requirements.txt, package.json).

Example workflow: /test-and-fix Run or add unit or end-to-end tests and fix the errors automatically to ensure code quality before committing, merging, or deploying.

Example workflow: /deploy-app Automate the steps to deploy your application to various environments (development, staging, production), including any necessary pre-deployment checks or post-deployment verifications.

Tip: Ask Cascade to create a workflow to automate a common use case: “Create a workflow to pull my Github issues, find the latest one, and execute on it.” After Cascade creates the workflow, you can try it out, and make edits based on common failure cases.

Tip: After completing a repeatable task with Cascade such as a series of terminal commands or file edits, you can prompt it to make a workflow: “Create a workflow to make this easier in the future.”

Tip: Workflows can take in arguments. For example, your workflow can say “If no PR link is provided, ask the user for one” and then you can run “/workflow <link>”.

Tip: Want to make sure Cascade auto-runs a command in a workflow? Add “// turbo” annotation above that command. If you want all commands to be auto-run, add "// turbo-all" anywhere in the workflow file. This will still respect any allow/deny-listed commands.
```
