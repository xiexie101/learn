# 500 - 《Devin》

> 感谢群友 @Erhu 的账号，下午试用了下，写下感受。

1、Devin 官方对自己的定义是「Devin is a collaborative AI teammate」。我用下来，感觉他是 Computer Agent，和 Cursor、Windcurf 有重合，但不算一类。感觉 Devin 的特长倒不是编程，对于专业程序员来说，他的编程流畅度也没有在本地使用 Cursor、Windcurf、Github Copilot 等要好。但，Devin 是一个基本可以做任何事的 Agent，虽然不一定做地有多好。

2、试过这些任务。

1）[给他一个任务，让他提 pr](https://github.com/test-devin/test-devin/pull/1)  
2）review pr，给出修改建议后，让他调整  
3）[让他回复 issue 并考虑关闭](https://github.com/test-devin/test-devin/issues/3)  
4）[让他实现一个 issue 描述的能力](https://github.com/test-devin/test-devin/pull/4)  
5）让他打开一篇微信文章并保存内容到 markdown 文件  
6）[让他在 tnf 框架的基础上新增一个 todos page](https://github.com/test-devin/test-devin-tnf/pull/1)  
7）[让他为 tnf 新增一个复杂 feature](https://github.com/umijs/tnf/pull/87)

3、这应该是我目前见过功能最完善的 Agent 了，虽然主要功能是编程，但实际上是给了你一台电脑，你可以做大部分在电脑上能做的事。但是，离自动化完成编程任务还是有挺大距离。所以，就目前 Devin $500/m 的价格来说，一个月 250 ACUs（ACU 是 Devin 内部的计量单位），招一个初级程序员其实比他更划算。

4、[Devin 官方文档也有说他擅长做什么和不擅长做什么](https://docs.devin.ai/get-started/devin-intro#what-are-devins-strengths)。他擅长的是相对规范、可验证、初级任务、小改动和有好的例子供学习的任务，不擅长大型、一致性和可靠性、UI 好看的任务。这和用其他 AI Copilot 工具的体验是差不多的。

5、Devin 通常是以仓库为工作单位的，你需要初始化一个 Github 仓库（目前还不支持非 Github 仓库）。初始化需要 4 步，你需要确保几件事，1）安装好初始依赖，2）告诉他怎么更新代码（通常是 git pull）和更新依赖（通常是 pnpm install），3）告诉他 Knowledge，Devin 里叫 Knowledge，就是关于这个项目的信息，比如目录结构、实施要求、代码风格等，和 Cursor 里的 .cursorrule 类似（注：Knowledge 会随着任务的进行逐渐补充）。

![](https://tcc.sorrycc.com/p/5eb98dad-c4a0-463f-b5be-200969106b05.png)

6、以下是工作界面。左下是输入框；右边是「Devin’ Workshop」，分了 Shell（命令行）、Browser（浏览器，比如打开 dev server 或网页登任务）、Editor（修改或新增的文件会出现在这，但不能手动修改）、Planner（计划，很重要）。

![](https://tcc.sorrycc.com/p/bd4359cd-638d-4ee0-9735-57852e8fbdff.png)

7、当收到一个任务后，Devin 会先做计划。有时会和你确认计划。遇到他拿不定主意的想法时，也会和你确认。当任务模糊不清时，会要求你补充关于具体一些点的详情。遇到有风险的任务时，也会和你确认。

![](https://tcc.sorrycc.com/p/0d6d9442-f7c8-4f09-a827-64ceda2ec5ae.png)

他还会在 notes.txt 里写当前任务相关的需求、实现和参考等。

![](https://tcc.sorrycc.com/p/2e3a5b9f-a1e8-4bf8-b1d7-9c0e826e30a3.png)

[他会自动修 CI。](https://github.com/umijs/tnf/pull/89)你也可以要求他在提交之前执行某些命令。

8、补充一些感受。1）我一直希望有这么一个 AI Agent 出现，就是他太贵了，并且我之前也尝试过，Devin 新闻刚出的时候，我也试着手动写了一个，见 [MDH Weekly 123 – 《AI 程序员》](https://sorrycc.com/mdh-123/)，但水平和时间有限，只实现了个极简版，2）Cursor 和 Devin 怎么选？各自做擅长的吧，Devin 可以做 Agent 擅长的事，不仅是编程，当然编程他也不太弱，简单的编程任务可以交给 Devin，比如写了个 issue，然后 @ Devin 让他提 pr 实现了，但对于复杂的编程任务，还是本地用 Cursor 或其他 Copilot 工具更顺手，因为 Devin 跑起来相对还是比较慢的，3）把自己都没想好需求的任务交给 AI，我是没把握的，4）值得入手吗？不值得，太贵了，不值这个价，用 Cursor 就够了。
