# 605 - 《我怎么用 AI 编程 2025.11》

1、最近一直是 Cursor + Local CLI Code Agent + Remote Code Agent 的组合。不过 Cursor 只用 Tab Tab 自动提示的功能，写代码主要依赖 CLI Code Agent，小的琐碎的临时起意的任务会交给 Remote Code Agent。

![](https://pic.sorrycc.com/proxy/1763046522400-416396521.png)

2、CLI Code Agent 主要用两款，Claude Code 和 [Neovate Code](https://github.com/neovateai/neovate-code/)。有时在 Cursor 里跑，有时也在 iTerm 2 / Warp 里跑。之前主要用 Claude Code，现在则 80% 代码用 Neovate Code 写的。我是 Neovate Code 作者，所以吃自己狗粮，Write Neovate Code with Neovate Code。之前吃着吃着会回到 Claude Code，现在不会，已经好多天没开 Claude Code 了。另外也是 Neovate Code 新加的 BrainStorm 模式太好用，需求理的很清楚。

![](https://pic.sorrycc.com/proxy/1763046977683-342384253.png)

3、Remote Code Agent 主要用 Claude Code 官方 Web 版，支持手机端。体验和本地的 Claude Code 一致，而且支持 teleport 到本地。

![](https://pic.sorrycc.com/proxy/1763083598214-388533068.png)

4、CLI Code Agent 的 Workflow 是 Parallel Code Agent + Spec Driven 的组合。前者基于 Git Worktree，但可选；后者除非特简单的需求，否则必选，先脑暴产出 Design 方案，然后做实施。完整流程是「Workspace Create → Brainstorm → Save Design → Implement → Review → Commit → Workspace complete」，当然，灵活使用，不会所有需求都完整把流程走全。

> 注：这一套也可以用于 Claude Code 里。如果是 Claude Code，推荐用 conductor.build 做 workspace 管理，然后加一套 slash command 做 brainstorm、save design、implement、review 和 commit 等。

5、我目前用的 Claude 的 Pro 订阅，编码主模型是 Sonnet 4.5，辅模型就多了，包括国产的和 Openrouter 上的免费模型和 Gemini 等。主模型量用完后会用其他的。虽然用的 Claude 的订阅，但用的是通过 claude-relay-service 把 api 代理出来给 Neovate Code 用的方式。

![](https://pic.sorrycc.com/proxy/1763046763244-539516761.png)

6、由于 Pro 订阅 Limit 有限，所以在长时间编码阶段，会用 Sonnet 4.5 产出 Design 方案，再用其他模型做实施，效果也不错的，而且国产模型速度快，跑起来省时间（2025.11.18 更新：优先用 Cursor、Kiro 的 Sonnet 模型做实施，因为有买会员或者有免费的 Credit，目前量够，用完再考虑国产模型）。Sonnet 4.5 模型不够用时，也会临时用 OpenRouter 的 Sonnet 模型作为后备。国产模型推荐 [iflow.cn](http://iflow.cn)，除了新出的那几个之外，之前的大多国产模型都可以免费用。

![](https://pic.sorrycc.com/proxy/1763046620023-67817690.png)

7、由于做 Code Agent 的关系，所以大多 Code Agent 的 CLI 和 IDE 都会用用，大多都模型提供商的服务也都有注册着。

![](https://pic.sorrycc.com/proxy/1763045999495-576265446.png)
