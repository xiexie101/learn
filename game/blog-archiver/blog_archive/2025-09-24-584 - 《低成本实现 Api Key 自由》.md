# 584 - 《低成本实现 Api Key 自由》

> 介绍我如何购买和使用 Claude、Google、OpenAI 和国产四类主流 AI 模型。

1、要想 API KEY 自由，尤其是程序员的场景，你得拥有这四类模型的使用，分别是 Claude、Google、OpenAI 和国产模型。主编码用 Claude、辅编码用 Google 和 OpenAI，琐碎任务用国产模型。国外好的模型用着省心一些，但国内主流开源模型也已经能用了，只不过你需要额外多花一些心思写好 prompt、组织上下文和裁剪任务。

2、现在分别介绍我如何用各家模型。

1）Claude。

用 Apple Store 注册，可美区，也可参考 [583 - 《尼区 Claude 续》](https://sorrycc.com/nigeria-claude-2) 用尼区账号。尼区稍微便宜点，¥90 块钱以内一个月普通会员，¥600+ 一个月是 $100 的会员（带 opus）。然后用 [claude-relay-service](https://github.com/Wei-Shaw/claude-relay-service) 搭一个，走美区家宽代理，就可以通过 token 的方式用了，还可以分享给你的朋友一起用。

2）Google。

Google 免费账号就有 Pro 和 Flash API KEY 的权限，所以直接搞到 Google 账号，然后创建项目和 API KEY 即可。一个 Google 账号可以到 [https://console.cloud.google.com/](https://console.cloud.google.com/) 建 12 个项目，然后到 [https://aistudio.google.com/](https://aistudio.google.com/) 上给每个项目建一个 api key，再用 [https://github.com/snailyp/gemini-balance](https://github.com/snailyp/gemini-balance) 分配，基本上就够用了。（注：Free Tier 的 google 账号，Limit 是按项目分的，所以不要一个项目建多个 api key，没用）

限制如下，1）Gemini 2.5 Pro 5 RPM、250,000 TPM、100 RPD，2）Gemini 2.5 Flash 10 RPM、250,000 TPM、250 RPD。以 2.5 Pro 为例，12 个项目的 key，每分钟 60 个请求，每天 6000 个请求，基本够用，不够的话搞 2-3 个 google 账号就好了。

另外，最近比较多 503 overload 的报错，遇到了可以多加一些 IP 代理来绕过。

还有，如果遇到 429 的问题，应该是账号所在区域和访问 IP 不一致导致 API Key 被封掉了。换一个美区的 Google 账号吧。

3）OpenAI。

闲鱼搜 codex，然后可以 8 - 10 块钱买到。给他邮箱，然后把你的账号加到一个 team 里，登录后就可以用 plus 相关的各种功能了。包括 web、codex platform 和 codex cli。

然后把账号加到 [claude-relay-service](https://github.com/Wei-Shaw/claude-relay-service)，就可以通过 api 的方式给其他场景用。（注：这里是 response api，有些 client 不兼容这种）

4）国产模型。

选择一是用 [https://iflow.cn/](https://iflow.cn/) ，目前免费，参考 [https://neovateai.dev/zh-CN/docs/providers/#iflow](https://neovateai.dev/zh-CN/docs/providers/#iflow) 。阿里的服务，注册后拿到 api key 就可以用各种国产开源模型。包括阿里的 qwen 3 max，这是我目前干杂活用的比较多的模型。

选择二是用 [https://modelscope.cn/](https://modelscope.cn/) ，github 登录、接码，每天 2000 个免费请求数。缺点是没有 qwen 3 max 。
