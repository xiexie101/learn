# 547 - 《AI Tips（6）》

> Grok 3 API、AutoRAG、神秘模型、Vibe Coding、我的语音输入方法等。

1、**[x.ai](http://x.ai) 终于上 Grok 3**，可以把之前花 $5 兑的几个 $150 每月的账户给用起来了。它拆了几个模型，都支持 Function Call，5rps，上下文 131K。如果不知道区别，可以直接无脑用 grok-3-fast-beta。同时，我把本地的文章总结接口换成 grok-3-fast-beta 了，试过效果还不错，后续 [2025-04](https://sorrycc.com/readings-2025-04) 这系列的会先 Grok 来做总结。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/eI2g0r.png)

2、**[firebase.studio](https://firebase.studio/)**，类似 v0.dev 的，基于 Gemini 模型，支持一键部署到 Firebase App Hosting，实现从开发到上线的一站式体验。但，试了下，应该不适合我。。不过它的流程其实可以参考一下，当你提出一个需求的时候，它会先给一个设计方案，然后你确认这个上级方案没有问题之后，他才会再接着往下去实施。

3、**AutoRAG。** cloudflare 的 autorag 还挺有意思，基于 r2，只要把文档同步到 r2 里，它就会做自动的 RAG。我试着把我所有的个人博客传上去，试了一下，效果还不错。我把它接到了博客上（右下角的那个气泡），并且持续更新文档的内容。同时感觉如果企业内要做知识库的系统，这种方式也挺好的，比较白盒。就是平台方提供一个空间，然后开发者负责把文档传上去（注：cloudflare 后续还会支持 web crawler 和 d1），平台会自动对你这个空间里的文档和资料去做 rag，然后基于这个空间提供对外的 API 接口去做 search 和 ai search。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/iVkHXP.png)

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/minicc.ask.png)

步骤如下：

1）新建 r2 bucket，上传文件（注：也可以通过 wrangler r2 object put 上传）  
2）创建 autorag，指定前面创建的 r2 bucket，会自动开始索引  
3）在 autorag 的 playground 验证效果，同时也可以点「Use AutoRAG」通过 cloudflare worker 或 api 的方式使用他，他提供了 search 和 api search 两种类型的接口

```bash
wrangler login
wrangler r2 object put bucket_name/foo.md --file foo.md --remote
wrangler r2 object delete bucket_name/foo.md --remote
wrangler r2 object get bucket_name/foo.md --remote
```

4、**[Wordpress 的 AI Website Builder](https://wordpress.com/ai-website-builder/)**。试了一下，它报错，没有办法继续。。

5、**两个神秘模型，[quasar-alpha](https://openrouter.ai/openrouter/quasar-alpha) 和 [optimus-alpha](https://openrouter.ai/openrouter/optimus-alpha)，免费。** 1M 上下文，32K 最大输出。免费用户每天 50 次调用，账户上超过 $10 用户每天 1000 次调用。据说后者的 SVG 能力不错，媲美 Claude Sonnet 。

6、**[Augment Code](https://www.augmentcode.com/)**，Cursor 竞品。网上有人把他夸得很厉害，但是感觉可能不是真心夸的。我之前就注册过，但一直没有试。他有个和其他方案不一样的点是，他会把你所有的代码都上传到服务器上，然后在服务器上去对这些代码去做索引，优缺点都有，看怎么权衡。

7、**不用 Cursor 照样 Vibe Coding。** 早上试了一下，先用 Google 的模型做计划，提出需求，他会问我一些问题，我补充信息之后，他会产出一份详细的实施文档。确认一下实时文档，如果需要调整的就手动改一下文档。再用 Claude 的模型去实施方案。（过程中我还试了下用 Google 的模型去实施，效果没有 Claude 的好）。

视频见 [https://g1i4rxq8hc.feishu.cn/docx/OB9IdyIVboLjlZxvWYVcgAkanJf](https://g1i4rxq8hc.feishu.cn/docx/OB9IdyIVboLjlZxvWYVcgAkanJf) 。

8、**语音输入。** 随着模型能力的提升，现在语音输入的精确性越来越高了，我现在大量依赖这个能力去做输入，包括写文章、编码等。Mac 上我现在有三种语音输入的方式，1）Wispr Flow，之前有介绍过，2）豆包客户端的语音输入功能，3）Whisper 闪念笔记 Shortcut，在 [541 - 《AI Tips（5）》](https://sorrycc.com/ai-tips-05) 里介绍过。豆包的语音输入的中文能力比较强，但是对英文的处理稍微弱一点，所以我会有选择的去使用，当有需要输入到英文夹杂的内容时，更倾向于用 Wispr Flow，其他场景用豆包。iPhone 上则是用前面的第三种，Whisper 闪念笔记 Shortcut 。
