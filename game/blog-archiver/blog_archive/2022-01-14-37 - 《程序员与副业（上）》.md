# 37 - 《程序员与副业（上）》

基于《Ten++ Ways to Make Money as a Developer》加上我的一些个人理解，介绍下程序员行业相关的副业。个人经验有限，如果存在不准确的地方，望指正。

**1、写博客**

为什么写博客？抛开副业和赚钱，主要还可以提升技能、深入 topic、营销自己提升影响力等。此外还要注意「持续」写。

博客可以基于 Gastby，部署到 Netfily，写博客就是创建 markdown 文件，push 到 github 后，过几分钟就完成部署，参考 [https://antfu.me/](https://antfu.me/) ；也可以用博客框架，比如 WordPress、Ghost、Typlog、BearBlog 等；还可以发布到既有平台，国外的比如 [dev.to](http://dev.to)、hashnode、freecodecamp，国内比如公众号。

内容方面，尽量选自己擅长的领域，注意不需要专家才开始写，因为总有人比你菜；选题方面分用户想读的和你想写的，后者优先，也可尽量找双赢的主题。

写完博客要推广，比如可以在社交平台 twitter、reddit、微博等，或者在朋友圈和群里转发，找到自己的观众。还有在其他平台写文章也是可以引流的，比如知乎、掘金、Bilibili、Youtube 等。如果你有写书、小册，也可以获得新的观众。

公众号通常有号主群。公众号之间可以做互推、加白、投稿等营销操作。互推据说是涨粉利器，但我还没试过。发文之后如果觉得有价值，可以在号主群里投稿，有人转的话也可以吸引部分观众。

找到观众后要留住观众和实现增长，内容当然是最重要的。还有个 Tip 是可以建立时间表，比如 2-3 篇/周，对自己有要求，让观众也有期待。

如果在平台上写，有一个问题是需要考虑的，如果被封号或者平台挂了怎么办？辛苦培养几年的观众群可能就打了水漂。作者给的解法是引导到 NewsLetter，邮件列表是平台无关的，同时相比社交媒体和博客平台，NewsLetter 更能触达用户。

NewsLetter 国外有免费的 [substack.com](http://substack.com)、[getrevue.co](http://getrevue.co) 和 [tinyletter.com](http://tinyletter.com)，付费的 [convertkit.com](http://convertkit.com) 和 [papyrus.so](http://papyrus.so)；国内有 zhubai.love 和 hedwig.pub。如果不想用平台，也可以用类似 [sendy.co](http://sendy.co) 或 [sendgrid.com](http://sendgrid.com) 自己发邮件，比付费平台便宜很多。

博客的赚钱方式比如广告、赞助文章，还可以推广自己的产品。广告注意别影响用户阅读，可以尝试换不同的地方投放，用短域名比如 [bit.ly](http://bit.ly) 统计点击量，然后观察哪个效果好；接赞助文章得小心，可能掉粉或者影响个人品牌，最好先问问自己这个产品你愿不愿意用，如果不是则不接，保持观众比赚块钱重要。国内公众号好像是倒着来的…

**2、1 对 1 辅导**

这个有一定门槛，需要先有影响力之类的。用金钱换时间和经验，可以是解决问题，也可以是帮助人成长、晋升或入职等。早早聊的业务之一，社区也有一些同学合伙在做，开课吧好像也有，属于课程的附带项目。

作者有个建议是可以从免费的开始，逐渐积累经验和人脉，做久了之后有一些成功案例的话，改付费会顺畅很多。
