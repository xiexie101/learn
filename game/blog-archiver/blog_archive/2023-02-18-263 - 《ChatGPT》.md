## ChatGPT 是什么

关于 ChatGPT 的资料有很多，其中这份 [A recap of ChatGPT | tech news - YouTube](https://www.youtube.com/watch?v=WVct5y3hBEg) 是我看过最有趣的。

![](https://img.alicdn.com/imgextra/i4/O1CN017lx1yk1XEaElBtKrN_!!6000000002892-2-tps-1744-978.png)

AI 为啥这么厉害？答案是 Transformer，他让我们构建 LLM（Large Language Model）的范式发生了变化。

![](https://img.alicdn.com/imgextra/i4/O1CN01AaU4Nz1nh0feArIJn_!!6000000005120-2-tps-1744-962.png)

Transformer 的论文其实是 Google 在 2017 年发表的，叫做「Attention Is All You Need」。其中有个优点是并行，相比 RNN 不需要一个词一个词地的处理，所以会让训练速度快很多。

![](https://img.alicdn.com/imgextra/i2/O1CN01Eb88op1MwaW4dzQZw_!!6000000001499-2-tps-1690-930.png)

然后就有了 Google 的 BERT 和 OpenAI 的 GPT。下图是每个模型的训练数据。

![](https://img.alicdn.com/imgextra/i4/O1CN01qwForu1TtZDLMH8VU_!!6000000002440-2-tps-2190-1130.png)

OpenAI 从网上拿数据，通过 Transformer 训练出 GPT-1。

![](https://img.alicdn.com/imgextra/i1/O1CN01YXa45324f2QtsfVsS_!!6000000007417-2-tps-2238-1186.png)

预训练的 Model 相比非预训练的 Model 的区别是，比如你要一个翻译模型和一个总结模型，非预训练的方式是从头开始做每个模型的训练，而预训练的模型是先训练一个模型，然后基于此微调成目标模型。

![](https://img.alicdn.com/imgextra/i3/O1CN01wYKzOC1cMR6Z4i6c5_!!6000000003586-2-tps-2216-1192.png)

总结下。1）Transformer 是模型架构，2）GPT-2 是 1.5B 的数据，GPT-3 是 175B 的数据，3）但是 GPT-3 的数据是未经打磨的，没有正确价值观的，他可能什么都会和你说，包括不合适的，而 GPT-3.5 是基于 GPT-3 进行的微调，告诉他什么是对的，什么是错的，4）现在你有了个乖巧的孩子，但你可能想要向别人炫耀他，所以经过打扮后就成了 ChatGPT 这个产品。

![](https://img.alicdn.com/imgextra/i1/O1CN01ToqYkc267GSVL8Sqf_!!6000000007614-2-tps-1486-954.png)

如果用七龙珠来做个比喻，悟空的头发就是模型的参数量。GPT-2 是 1.5B；GPT-3 是 175B，你可以看到悟空的头发变多了；而到了 GPT-3.5 或 ChatGPT，悟空的头发没有变多也没有变长，但是经过打磨后变得更漂亮了；最后到未来的 GPT-4，是 100T（100万亿）。

## 如何获取账号？

## 怎么用 ChatGPT？

1、最简单的用法就是直接打开 [http://ai.com/](http://ai.com/) （会跳转到 [chat.openai.com](http://chat.openai.com)），然后输入问题等回复。进阶点的用法是，1）下载 [lencx/ChatGPT](https://github.com/lencx/ChatGPT) 客户端，极小、基于 tauri 和 rust、有一些辅助功能比如 prompts 提示、注入用户脚本等，2）使用 Conversation（会话）功能，让 ChatGPT 记住上下文，但注意是有容量限制，记得是 4000 token（约 8000 字符），容量到了他就会忘记前面的。

2、你会问啥？比如

*   西游记第一回合讲了什么？
*   怎么比我的哥哥拿到更多家产？

*   请用「床前明月光」写个谐音梗
*   先有鸡还是先有蛋？
*   人生的意义是什么？
*   …

但是很明显，有些问题是得不到答案的…

## 基于 GPT-3 的应用

1、社区有大量基于 GPT-3 的应用，大部分是基于 [vercel 的 twitter-bio 模版](https://vercel.com/templates/next.js/twitter-bio)建的，在模版的基础上加入自己的 prompt 就是一个应用场景。比如有 [AI Commits](https://github.com/Nutlope/aicommits)、[周报生成器](https://weeklyreport.avemaria.fun/)、[README 生成器](https://readme.rustc.cloud/)、[聊天简化器](https://chat-simplifier.imzbb.cc/zh)、[邮件助手](https://email-helper.vercel.app/)、[Teach you Anything](https://www.teach-anything.com/)、[Hey Smart Siri](https://github.com/Daiyimo/Access-chatGPT-in-Siri) 等。

2、这类场景有些要自己提供 TOKEN，因为作者如果是用 OpenAI 的免费额度，人一多的话，$18 很快就会用完了。这也是为啥前面说要备几个普通的 ChatGPT 账号的原因。

3、注意这类应用是 GPT-3 而不是 ChatGPT。ChatGPT 是 GPT 3.5，之前泄漏的模型是 text-chat-davinci-002-20221122；而 GPT-3 是 ChatGPT 的上一代，其模型是 text-davinci-003，能用，但没 ChatGPT 好用。

4、App = OpenAI API + Prompts。这类应用基本都是给用户输入的表单 + Prompts + OpenAI 的 text-davinci-003 模型。其实如果你有 ChatGPT 账号，更建议用 ChatGPT 的 prompts 加自己的输入，原因是 ChatGPT 的模型更好。

举一些例子。

*   AI Commits 的 [Prompts](https://github.com/Nutlope/aicommits/blob/68fc8ad/src/utils.ts#L65) 是「Write an insightful but concise Git commit message in a complete sentence in present tense for the following diff without prefacing it with anything:」
*   周报生成器的 [Prompts](https://github.com/guaguaguaxia/weekly_report/blob/151cb62/messages/en.json#L21) 是「请帮我把以下的工作内容填充为一篇完整的周报,用 markdown 格式以分点叙述的形式输出:」
*   README 生成器的 [Prompts](https://github.com/ycjcl868/readme-gpt/blob/997d1aa/messages/zh.json#L19) 是「请帮我把以下的 Github 项目介绍填充为一篇完整的 Github README 说明,用 markdown 格式叙述的形式输出,标题适当加一些 emoji 表情:」
*   聊天简化器的 [Prompts](https://github.com/zhengbangbo/chat-simplifier/blob/main/messages/zh.json#L18-L20) 是「请总结 A 与 B 之间的聊天内容。要求简明扼要，以一段话的形式输出」
*   邮件助手的 [Prompts](https://github.com/shengxinjing/email-helper/blob/main/pages/index.tsx#L37) 是「Generate a business email in English that is friendly, but still professional and appropriate for the workplace. The email topic is: …」
*   Teach you Anything 的 [Prompts](https://github.com/lvwzhen/teach-anything/blob/main/pages/index.tsx#L49-L52) 是「Explain … to a 6nd grader in English with a simple example.」或「Explain … in English in technical terms, divided into two paragraphs, principles and applications. Output format, Principle:, Application.」

5、推荐关注下 [yoheinakajima](https://twitter.com/yoheinakajima)，他是个投资人，同时还写了 20 多个 GPT-3 的应用，包罗万象。下面列一些我觉得有趣的（大部分都很有趣）。

*   AI Memo Drafting Tool。你输入一个 url 或者给一个 pdf，然后他会给出总结文本。实现思路应该就是提取内容，然后和总结类的 Prompt 一起喂给 AI。
*   [emails.wtf](https://www.emails.wtf/)。把邮件 Forward 给 xxx@emails.wtf，他会回复你总结的内容。我用 Prompt 「Please summarize this email and then translate your summary to chinese.」也可以实现类似效果。
*   Healing Poetry。如果你感到悲伤，让 AI 为你写一首治愈系的诗吧。手动 Prompt 可以这样，比如「Please write short poetry to healing me and then translate to chinese. I have too much work to do and not enough time.」。
*   [GTM Strategies](https://gtm.yohei.me/)。让 AI 提供 GTM（Go-To-Market）策略。盲猜 Prompt 是「Please give me 7 specific go-to-market strategies based on a startup description. I want to…」。
*   Sentiment Analysis of ProductHunt。基于 ProductHunt 的评论数据做情感分析。盲猜 Prompt 是「Summarize, then analysis of clusters of reviews by sentiment and topic, then give example comments of each cluster」。
*   LinkedIn Bio Generator。基于 LinkedIn 的数据生成 bio。我有个想法是，可以批量读候选人的简历，然后让 GPT3 总结下，并基于一些维度做个表格。

## ChatGPT API

由于官方并没有提供 API 的调用方式，所以社区有很多非官方调用 ChatGPT 的方式探索。基于此，大家就可以把 ChatGPT 接入微信、Telegram、飞书、钉钉、Hey Siri 等场景了。

1、ChatGPT 刚出的时候（2022.12）写过一篇 [228 - 《5 种 ChatGPT 的扩展方法》](https://sorrycc.com/5-way-to-extension-chatgpt)，感觉好多都已经过时了。

2、目前看下来，有 3 种使用 ChatGPT API 的方式。1）基于 GPT-3 接口，收费、效果没 ChatGPT 好，不推荐，2）基于有头浏览器（由于 CloudFlare 的限制，必须有头），免费，但普通账号有每小时一定量的 Rate Limit（Plus 账号未知），通常走浏览器的账号密码登录，或者手动提供 [TOKEN](https://chat.openai.com/api/auth/session)，3）基于 Chrome 插件，通过 Chrome 插件操作浏览器做接口响应。

3、基础库 Node 语言的有 [transitive-bullshit/chatgpt-api](https://github.com/transitive-bullshit/chatgpt-api) 和 [waylaidwanderer/node-chatgpt-api](https://github.com/waylaidwanderer/node-chatgpt-api)。这两个库的最新版都是基于 GPT-3 的模型，所以建议的方式是用前者的 V3 版本，这个版本基于 Browser，同时也可以用基于此的 [bytemate/chatapi-single](https://github.com/bytemate/chatapi-single)，包含额外的队列和代理等功能。Python 语言的可以用 [acheong08/ChatGPT](https://github.com/acheong08/ChatGPT)，其 V1 版本是基于 Browser 的。

4、中间有段时间应该有泄漏过一个「text-chat-davinci-002-20221122」模型，但现在已不能用了。

5、此外，还有基于浏览器插件的 [ChatGPT for Wechat](https://chatgpt4wechat.aow.me/)、基于 GPT-3 的 [bestony/ChatGPT-Feishu](https://github.com/bestony/ChatGPT-Feishu)。基于官方 API 实现的话，还可以用 [cfortuner/promptable](https://github.com/cfortuner/promptable)。

## 体会

1、一个体会是要「学会提问」。提问有一些技巧，同时也需要基础的积累，没有基础可能都不知道怎么问，「先有好问题才可能会得到好答案」。所以，**作为这个时代的工程师，需要有 3 种能力**，1）把业务问题转换成通用的软件问题，2）知道怎么问问题，3）知道应该接受怎样的答案，因为 ChatGPT 经常会一本正经的胡说八道。以及，要善用 ChatGPT、Copilot 来提效，帮我们处理枯燥乏味的那部分，比如查文档、写代码等。

2、视野 vs. 经验。我们可能拥有很多经验，而 ChatGPT 拥有的是视野；就像我买一些付费的知识产品，其实也是为了拥有更好的视野。

3、ChatGPT 能代替 Google 吗？哈哈。
