# 282 - 《我的小技巧集合 02》

> 每凑够 10 条发一篇。

*   如何 Profile Node 应用？1、跑 Node 时加 `--cpu-prof` 参数，比如 `node --cpu-prof $(which npm) run version`，结束后会产出一个或多个 `.cpuprofile` 文件，2、打开 [https://www.speedscope.app/](https://www.speedscope.app/) 上传文件即可查看火焰图，包含详细的堆栈信息。参考[这个](https://t.me/yqtalk/184)和[这个](https://marvinh.dev/blog/speeding-up-javascript-ecosystem-part-4/)。
*   一个可以[解锁任意网站「复制」限制的小技巧](https://t.me/yqtalk/296)。步骤是，1）打开 devtool，2）选中 body 元素，3）再右边选择「Event Listeners」把所有「copy」事件 remove 掉。（注：评论区有更简单的方案）
*   减少 Round Trip（往返）是可以提效的。举两个例子，1）执行命令时有两个选择，a> npm i && npm run build 同时敲，b> 先敲 npm i，等他完成后再敲 npm run build，明显前者效率会更高，比如我今天（2023.03.31）答疑时就应用了这个策略，在复现别人问题时的常用命令组合是 `git clone xxx && cd xxx && tnpm i && tnpm run build`，一次性敲完然后过会再去看他有没有报错，2）交代别人任务时或者提问时，把各种可能的情况都说清楚，如果出现 xx 情况应该怎么做，不用等对方回复之后再接着问，这样可以减少来回沟通的次数，可以提高效率，比如有些人通常会先问你好，然后寒暄，然后说问题，可能一次还说不清楚，更好的方式是一次性把问题讲清楚，以及包括各种分支应该如何处理，如果对方在怎么样，如果对方不会某件事应该怎么样，用程序员的话说，就是把 if else 描述清楚。
*   日常任务 SOP（标准作业流程）化，让任何人或者非专业用户都能操作，然后就可以找其他同学协助处理了。近期我会整理各种 SOP，包括个人的和工作的，工作的比如答疑 SOP、发布 SOP，个人的比如信息获取 SOP、写作 SOP，等等。
*   付费获取资源，免费的才是最贵的。最近看到有价值的群和产品基本都无脑入了，感觉能学到一点就赚了。
*   一个 Twitter 搜索技巧，在列表的基础上加各种 filter，包括最低 fav 数、时间、是否包含链接等，比如大家搜 `list:1462723878950690817 since:2023-03-29 min_faves:200 filter:links`，即可包含近期所有的前端资讯。
*   如果你微信公众号发文，推荐用 [MDX Editor | 一个微信排版编辑器，使用 MDX 来排版](https://editor.runjs.cool/create)，mdnice 以及我自己写的工具，最近微信公众号平台更新后在保存时会刷新格式，导致不少格式乱掉，这个 MDX Editor 是唯一遇到没有问题的。
*   一个小 Tip，随时带「指甲刀」，永远保持合适的指甲长度敲击键盘。 来源： [https://www.craft.do/s/LDWv7CqT53t2Ac](https://www.craft.do/s/LDWv7CqT53t2Ac)
*   又一个小 Tip，大家可以想一下，你过去一周，你觉得什么事情会让你经常想起来要做，会卡主你的，现在马上花 10 分钟去干这件事情，会有奇效。很多时候其实是冲动一下就能干的，有些事情其实想不清楚，跟自己说，接下来 10 分钟我必须立刻启动这个事情，很多事情就是这么启动出来的。行动的价值，哈哈。
*   投时间和钱研究科学上网、Github Copilot（还有 [GitHub Copilot Labs](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-labs)）、ChatGPT，ROI 绝对高。
