# 253 - 《工作目标 2023》

和大家分享下 Bigfish/Umi 团队今年的目标（暂定，未最终确定），希望可以给其他做基础技术的同学一些思路。如果你愿意，也欢迎告诉我你的目标，可以留言，也可以私信我。

今年没有框架层大的变动，目标包含三个主要方向，最佳实践、Bigfish x 业务、构建提速，以及两个辅助方向，答疑提效、Bigfish 课程。

下面是三个主要方向。

1、最佳实践。是什么？最佳实践就是做一件事最推荐的方式，比如如何写样式、要不要用 TypeScript、用哪个 CSS 方案等等。为什么做？1）团队代码一致性，2）挑选使用更适合我们的先进方案，提升 UX 的同时，也提升 DX。为啥能提升 DX？比如用 react-query/swr，相比用 axios/use-request，有些方案的天花板可能是另一些方案的起点。去年已经做了 8 个子方向，在知识星球里应该都能找到相关文章。今年计划新增 10+ 子方向，包括 MOCK、组件研发、鉴权、OneAPI、官网、国际化、目录结构等等，都会有一次迭代。

2、Bigfish x 业务（alias：Bigfish 地气计划）。希望让 Bigfish 的同学离业务近一些，让 Bigfish 更接地气，用专业的技术，去解业务项目中的真实问题，该推的事情花时间去推。业务项目问题分两大类，DX 和 UX，包括启动热更流程构建慢、产物尺寸大、页面性能不好、可以优化的请求方案、以及其他细节 UX 问题。目前在推进 3 个业务项目，蚂蚁官网、蚂蚁招聘、蚂蚁内网。这块需要注意的是，别自己蒙头蛮干，出方案为主，然后找业务线的同学一起做。为啥？1）自己做速度慢，不能并行，2）你没业务同学懂业务，不知道哪里有坑。

3、构建提速。我们在这个领域目前有三驾马车，MFSU、ESMi、Turbopack（TODO）。今年在每个方向还会持续投入，但是到明年或者更后面，出于成本考虑，会只留一个持续深耕。与 Turbopack 相关的还有个个人的目标，就是学 Rust，目前已系统学完基础部分，在此推荐下 [Rust 语言圣经 - Rust语言圣经(Rust Course)](https://course.rs/)，讲地浅显易懂。

前两个方向都是离业务较近的，也是今年有意为之，因为脱离了业务玩技术，久而久之自己都会感觉有点虚。
