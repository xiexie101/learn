# 489 - 《vlt 和 vsr》

前几天翻到的，简单了解了下。

1、这两是一个团队背后的作品。vlt 是 package manager，类似 npm、pnpm、yarn；vsr 是 serverless registry，类似 npm registry。这个背后的团队比较厉害，有 npm 的作者 Isaac Z. Schlueter、Darcy Clarke 和 Node TSC 成员 Ruy Adorno。

2、Vlt 发音为「volt」，他在包管理、查询、工作区支持等方面都提供了比 npm 更强大和灵活的功能，同时性能也有所提升。

3、我觉得主要是查询，可以用 css query 的方式查某个 npm 依赖的上下级依赖情况，命令是 vlt query，还支持 gui 的方式查看。

比如。

```
vlt query "[name=semver]" --view=gui
```

![](https://tcc.sorrycc.com/p/78662525-1c51-458d-ae3e-70232b57c909.png)

更复杂的比如。

```
vlt query '[name="my-project"] > :is(:project > *[name="tap"][version="21.0.1"])' --view=gui
```

4、vsr 是 serverless registry，竞品有 verdaccio 和 jsr。他为公司提供了一种安全分发自定义 JavaScript 包的方法。它配备了一个 CDN、其他注册表的代理和其他你可能期望的东西，但最有趣的功能是细粒度访问令牌。它可以让你精确控制谁可以与你的软件包交互，以及他们如何交互。
