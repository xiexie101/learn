## 前端趋势

先下前端趋势。从不同的维度，看 2023 发生了什么，以及 2024 会发生什么，以及我的想法（可选）。

**1、前端框架。**

2023 年。Vue 发布了 3.3 和 3.4；Angular 发布了 16 和 17；React [翻新了文档](https://react.dev/)，发布了 [10 周年纪录片](https://www.youtube.com/watch?v=8pDqJVdNa44)，发布了 canary 但目前主要给 next.js 用（Vercel 也因其与 React 之间的特权关系而受到一些批评），除此之外没有其他正式发布；年初 Solid 和 Preact 引发的关于 [Signals](https://dev.to/this-is-learning/the-evolution-of-signals-in-javascript-8ob) 的讨论非常热闹；此外，还有 Svelte、Solid、Million、Qwik、htmx 也都值得关注。

![](https://img.alicdn.com/imgextra/i2/O1CN01CKlazI22xF3AtwsB1_!!6000000007186-2-tps-2388-1428.png)

![](https://img.alicdn.com/imgextra/i1/O1CN01wYbzjL1Iydvqir3dU_!!6000000000962-2-tps-796-722.png)

2024 年。预计 React 发布 18.x、[19 alpha](https://twitter.com/acdlite/status/1719474730363662473)、Offscreen 等，希望不只是美好的愿望；React Forget 虽然我非常期望，但由于其复杂度和投入度感觉今年依旧不会发布；Signal 在非 React 框架中继续推进，包括 Vue Vapor、Svelte 5、Angular 等。

![](https://img.alicdn.com/imgextra/i2/O1CN010MEidI1aICJEJxFvk_!!6000000003306-2-tps-1126-554.png)

我的想法。1）前端框架赢家通吃，大部分前端同学在三大框架中选其一深入就够了，此外的其他方案可能会火一段时间，但长久看都会被淘汰或流于小众，方案虽好，但没有生态就没有发展，2）三大框架首推 React，其在国内外的生态、岗位需求、下载量等都远超另外两个。（大家可以想想，为啥国内大厂普遍都不用 Vue？）

**2、元框架。**

2023 年。Nuxt.js 更新 3.x，发布 [Nuxt Devtool](https://devtools.nuxt.com/)；Next.js 发布 13.X 和 14，[引入 App Router](https://nextjs.org/blog/next-13-4) 和 RSC；Astro 发布 4 引入了 ViewTransitions 尝试让 MPA 复苏；Remix 切换构建器到 Vite。

![](https://img.alicdn.com/imgextra/i4/O1CN012F4VHW1wVrfxLxCzh_!!6000000006314-2-tps-500-756.png)

2024 年。React 社区的元框架纷纷跟进 Next.js 支持 RSC，比如 [Remix 支持 RSC](https://twitter.com/ryanflorence/status/1729274387671760936)，[Redwood 支持 RSC](https://tom.preston-werner.com/2023/05/30/redwoods-next-epoch-all-in-on-rsc)，[Docusurus 支持 RSC](https://github.com/facebook/docusaurus/issues/9089)；大多开发者也将学会使用 RSC；Umi 发布 5；Astro 和 Remix 支持 SPA；其他还有 Waku 值得关注。

我的想法。

1）RSC 是把双刃剑。优点是快，我们的 H5 也用了，缺点就比较多了，比如研发成本、开发体验、适用场景、生态兼容、框架适配、部署复杂度等。都是权衡，就看快这件事是否足够重要和值得投入了；

![](https://img.alicdn.com/imgextra/i4/O1CN01qYyFvL1IaMwC4cPrH_!!6000000000909-0-tps-1286-924.jpg)

2）MPA 昙花一现，用户体验始终还是比不上 SPA，Astro 和 Remix 两个 MPA 的框架都在考虑支持 SPA。

![](https://img.alicdn.com/imgextra/i2/O1CN01LVFPDg1Bx5GvmI0uP_!!6000000000011-2-tps-3840-2160.png)

**3、React 生态。**

2023。请求方案，TanStack Query 发布 5；CSS 方案，Meta 的原子化 CSS in JS 方案 stylex 开源，TailwindCSS 发布 3.3 和 3.4；路由方案，React Router 更新 6.X；数据流方案，xstate 发布 5；组件库方面，shadcn/ui 一枝独秀。

2024。感觉变化不会大，已经过了百花齐放的时代，问题都已解地差不多，没那么多问题自然就没那么多方案；大批类 shadcn/ui 组件库发布；RSC 和现有的请求方案是冲突的，有匹配 RSC 的请求方案出来，或者现有请求方案兼容 RSC。

我的想法。1）shadcn/ui 之所以成功，除了文档、代码质量和组件丰富度，还有一点不同是，他并不是和其他组件库一样，以 npm 包的方式提供，而是通过 CLI 工具，直接复制源码到用户项目中，以及安装必要的依赖，这给了用户对代码更大的掌控权，2024 应该会有更多类似的组件库出来（几年前我们做 Umi UI 的时候也做过这种，但没有推广开放弃了）。

**4、构建工具。**

2023。Vite [发布 5](https://vitejs.dev/blog/announcing-vite5) 并官宣[用 Rust 开发 Rolldown 代替 Rollup](https://twitter.com/youyuxi/status/1709943106215530867)；Turbopack 在 Next.js 的 Dev 下可用；Farm 和 Rspack 发布；Mako 研发完成并在内部发布。

![](https://img.alicdn.com/imgextra/i4/O1CN01tOZyiH25ljwfkAS2x_!!6000000007567-0-tps-1024-1024.jpg)

2024。Rolldown 上半年发布；Turbopack 在 Next.js 默认开启并支持 build；Vite 下载量超过 Webpack；Mako 开源。

我的想法。1）大厂才有动力和资源做基于 Rust 的构建工具，2）Vite 发展真猛，相比 Webpack 的下载量[从年初的 1/12 减少到 1/3](https://npmcharts.com/compare/vite,webpack?interval=7)，新兴框架大多使用 Vite，但大厂框架除外，原因可能有二，a）bundless 带来的 dev 和 build 不一致会给生产带来不确定性，b）对于巨石应用来说，vite 在构建和运行时不一定快。

**5、前端工程化。**

2023。ESLint 引入 Flat Config；Prettier 发布 3；Biome 赢得 Prettier 挑战的 $22.5K 奖金；Bun 发布 1（放到这个类别可能有些歧义，但对我来说他就是）；Yarn 发布 4；Pnpm 发布 8。

![](https://img.alicdn.com/imgextra/i2/O1CN0176ryVT1safMU8IuXy_!!6000000005783-0-tps-3072-4096.jpg)

2024。Rust 占据更多前端工具链的市场份额；Biome 占据部分 Prettier 市场；Oxc 占据部分 Eslint 市场；将有 Rust 版的 npm 包管理工具发布，不一定是 [pacquet](https://github.com/pnpm/pacquet)；ESLint 发布 9；Prettier 发布 4。

我的想法。2024 前端工程化是 Rust 的。1）所有能用 Rust 重写的都将用 Rust 重写，包括构建、Lint、Prettier、测试、编译器、元框架等等，只要有「利」可图，比之前快，就有人会做，2）更多的人学会 Rust，同时随着 Rust 编译器的不断完善和更多学习资源的出现，其学习成本会继续降低。

![](https://img.alicdn.com/imgextra/i1/O1CN01pBYXAB1cyuTccxKMS_!!6000000003670-2-tps-1046-836.png)

**6、AI。**

2023。GPT 4.5、GPT 4 Vision、Github Copilot、Copilot Chat、[devv.ai](https://devv.ai/) 等的发布和迭代，对程序员来说都是提效利器，同时也降低了学习新知识的成本；[v0.dev](https://v0.dev/) 和 [tldraw/make-real](https://github.com/tldraw/make-real) 对前端有里程碑意义。

![](https://img.alicdn.com/imgextra/i1/O1CN01gsB00u27O2gW1h7e4_!!6000000007786-0-tps-554-554.jpg)

2024。GPT 5 发布；AI 让程序员进一步提效，低水平工作 AI 化，市场不再需要那么多初级程序员，影响就业；NoCode/LowCode 平台用 AI 代替拖拽式 UI，比如 [Basedash](https://www.basedash.com/)；框架和 AI 也有会更多的结合点，比如 [Million Copilot](https://copilot.million.dev/)，AI 可以告诉你为啥应用跑起来慢。

![](https://img.alicdn.com/imgextra/i1/O1CN01aXcCeB1IqOniMmw5E_!!6000000000944-0-tps-906-882.jpg)

我的想法。1）AI 已经是继科学上网后又一项程序员必备技能，当然也可以不学，只是效率低 5 倍而已，见图，最好会用，并且用熟练，但是也不要期望能更早下班，相信老板，他会把你空出来的时间填满的。

![](https://img.alicdn.com/imgextra/i1/O1CN0156UcNd1cffqBTDVDg_!!6000000003628-0-tps-1290-1655.jpg)

2023。Docusaurus 发布 3。Safari 发布 17。更多参考 [https://mdhweekly.com/weekly](https://mdhweekly.com/weekly) 。

## 我们为什么要了解趋势

在回答“为什么”之前先问“是不是”。

1、我们是不是要了解趋势？我的理解是。有两类开发者，a）为其他开发者构建工具、库和其他产品的人，b）使用上述产品构建面向最终用户的应用软件的人。如果你是前者，那关注趋势可以带来更好的视野，少踩坑；而如果你是后者，那我觉得关注趋势也可以带来更好的视野，但是，你通常会有更高优先级的事可以做，比如「基础、书和深度好文」，看这些东西，会相比关注趋势更有价值。

2、当我们在关注趋势时应该关注什么？业务。

## 我的 2024

> 分享下我在 2024 的规划，如果大家还没订计划，希望对大家有所帮助。

1、精通 Rust、RSC 和 AI。这三个是基于趋势和个人工作任务之后，我觉得今年最应该投时间进去的技术点。Rust 主要是用于开发工具和框架；RSC 是框架的趋势，不仅是学会使用，今年还需要让 Umi 也支持 RSC 的研发方式；AI 持续跟进，个人对效率有些偏执的追求，所以会去尝试各种可能的方案。

2、多看基础、书、深度好文，少看趋势、新闻、八卦。比如刷各个语言和框架的官网文档、手册和书。今年我在整理 mdhweekly 年终总结时，趋势类和新闻类的内容是直接略过的，因为回头看，其实价值并不大。

3、多输出、重健康。多写作，我在某个平台写了 397 篇，个人收获颇多，字重要的是「记」得住了，写过的人自然会有体会。

![](https://img.alicdn.com/imgextra/i4/O1CN01mYa2GK1sqhE5lgrdR_!!6000000005818-0-tps-1525-934.jpg)

4、工作上开源 Mako、发布 Umi 5。
