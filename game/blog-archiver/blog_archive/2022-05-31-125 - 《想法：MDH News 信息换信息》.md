# 125 - 《想法：MDH News 信息换信息》

先分享几个失败的小故事，

1、很久以前，我组建过「MDH：情报一组」微信群，希望大家能互通前端资讯，然后分享的人寥寥无几，最终沦为闲聊群

2、更久之前，github private org 免费之后，我利用 github ci 搭了个早鸟群来激励大家写文章，通过 hp 方式来做约束，hp 小于 0 时退出 org，最终我被自己的程序清退，今天想找下是哪个 org 都找不到了（感谢小虎提醒：[https://github.com/earlyBirdCamp](https://github.com/earlyBirdCamp) 和 [https://earlybirdcamp.sorrycc.com/](https://earlybirdcamp.sorrycc.com/) ）

再分享看到的成功故事，

1、某周刊有个「Give Me Five」的活动，出 5 个题目，一定时间内，邀请大家分享各自关于这些问题的答案，时间截止后，参与的同学会收到所有人关于这些问题的答案

然后周末在看完 Remix 的视频教程后想着写点啥，就又想到了这个。思路还是一样，用游戏化的方式做前端信息共享的小圈子，利用 HP（血量）来激励和敦促大家进行分享，以达到「用信息换信息」的目的。

基础功能如下。

1、初始 HP 7（不输出也可以活 7 天）  
2、每分享一则信息加一定的 HP，数值根据分享信息的质量和类型而定  
3、HP 每天 -1，如果 HP = 0，则不再提供信息访问权限  
4、通过 Github 登录

如果能活下来，会做的高级功能。

1、提供 RSS 订阅服务  
2、支持「赞」和「踩」，赞 >=7 时，HP 额外 +7，踩 >=7 时，HP 额外 -7  
3、异常账号比如分享了 RSS 链接的会被封禁  
4、分享过的 URL 不能多次分享  
5、不定期提供「HP 兑换礼物」服务  
6、HP 归 0 依旧可访问自己的历史内容

Q：加入这个圈子，可以收获啥？

1、高质量的小圈分享（预想）  
2、多写多看，有更多输入，同时有更多输出
