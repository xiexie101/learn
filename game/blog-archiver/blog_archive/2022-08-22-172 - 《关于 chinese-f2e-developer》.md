# 172 - 《关于 chinese-f2e-developer》

> 水一篇，本文明天同步更新到公众号。

仓库地址是 [https://github.com/sorrycc/chinese-f2e-developer](https://github.com/sorrycc/chinese-f2e-developer) 。

上周转载了 Yingci 的文章「国内 FE 圈常见用语方法论」，发现大家的关注度挺高，因为这是我公众号里留言最多的一篇。。然后上周五又看到「chinese-independent-developer」仓库，于是手痒整理了「chinese-f2e-developer」，希望能网罗国内前端开发者，但时间有限，第一版主要是我视野范围内的前端。

github 上都说明了「个人视野有限，请不吝 PR！缓慢施工 🚧 中」，可能有些同学太心急，或者是可能名字取太大，触碰到不少同学敏感的心，在此说声抱歉。个人接触面较窄，关注框架、工具和 React 社区的更多一些，国内文章看得少，知乎逛地也不多，才发现漏了不少知乎上和其他领域的前端大佬。知乎相关问题 PV 24W+，不过这数据有点假，因为引流到 github 的只有 300+ PV，连零头都不到。

回顾下为啥整理一个列表都会被骂。名字取太大？观众自己或关注的人不在列表里？其实可以理解，当时看到开源中国那个榜单里没我，我也在心里骂过。本想叫「前端英雄榜」来着，现在看来，幸好是没取这个名字，不然被骂更惨。

那为啥整理这个列表？~~因为我爱分享！关注我的同学会发现我还整理过「awesome-tools」、「awesome-javascript」和「awesome-f2e-libs」，同时希望让更多同学了解前端社区，尤其是初学者，解「不知道关注谁」的问题。~~因为我想给我的「知识星球」带带货，顺道刷点 Github Contribution，因为 Review PR 刷起来最快，你看，那天的 Github 色块都快黑了！于我来说，一个额外收获是认识了很多之前不认识的同学。

不知大家平时看文章或者库会不会关注背后的作者？我会。看文章、翻源码、看书之前，我都会先了解作者是谁，有啥经历，作为是否继续花时间深入的考虑因素之一。比如「Josh W Comeau」的文章，我就每篇都会细看。所以，这也是这个列表的目的之一，希望大家先找到靠谱的人，再去看他的作品和更新。

列表上有哪些人是大家争议最大的点，爱贴标签的朋友们又轻轻地掏出了「前端娱乐圈」，令人费解。有人娱乐，有人痴迷技术，有人写文章，有人录视频，有人经营公众号，有人搞直播，有人卖课，有人虽技术不好但能把复杂问题讲清楚教给初学者，有人虽不在大厂但能开班引导大量学生进大厂。这不是很正常么，大家都在解决问题。沉心搞技术和开公众号写入门文章哪个更有价值？还真说不准。
