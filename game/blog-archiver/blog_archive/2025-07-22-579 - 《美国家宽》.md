# 579 - 《美国家宽》

> 感谢群友提供的信息。

1、背景是公司的网络不能再用之前的翻墙服务，于是我需要在两个机场之间来回切换。 但是担心 IP 的频繁变更会让 Claude 把我的号封了，于是花 $2 不到买了个美国家宽 IP 。检测结果见下图。这下可以安心用 Claude 了。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/image-2025-07@main/uPic/Google%20Chrome%202025-07-22%2012.39.24.png)

2、购买渠道是走的 [webshare.io](https://www.webshare.io/?referral_code=w4zxzx7kqt51)，登录后到「Plans & Pricing」页，选「Static Residential」、「Private」、「2 Proxy」、「250GB/month」，Location of Proxies 选「United States」，一个月 $2 不到。

3、买完之后回到 [Proxy List](https://dashboard.webshare.io/proxy/list) 就能看到自己购买的两个 proxy 了，拿着 ip 地址去以下地址检测。

[https://www.ip2location.com/](https://www.ip2location.com/)  
[https://ping0.cc/](https://ping0.cc/)  
[https://ipinfo.io/](https://ipinfo.io/)

4、如果 IP 不好，在 Proxy List 页选「Action > Replace Proxy」换一个。

5、科学上网软件配置，以 Surge 为例。

```
[Proxy]
webshare-io = socks5, IP, PORT, USERNAME, PASSWORD, underlying-proxy=US

[Rule]
RULE-SET,https://ruleset.skk.moe/List/non_ip/ai.conf,webshare-io
RULE-SET,https://raw.githubusercontent.com/blackmatrix7/ios_rule_script/master/rule/Surge/Claude/Claude.list,webshare-io
DOMAIN,cdn.usefathom.com,webshare-io
DOMAIN-SUFFIX,anthropic.com,webshare-io
DOMAIN-SUFFIX,claude.ai,webshare-io
DOMAIN,servd-anthropic-website.b-cdn.net,webshare-io
DOMAIN-SUFFIX,anthropic.com,webshare-io
DOMAIN-SUFFIX,claude.ai,webshare-io
DOMAIN-SUFFIX,claudeusercontent.com,webshare-io
```

6、测试 Claude Code，正常。
