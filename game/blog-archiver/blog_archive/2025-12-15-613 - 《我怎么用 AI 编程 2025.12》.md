# 613 - 《我怎么用 AI 编程 2025.12》

距离写 [605 - 《我怎么用 AI 编程 2025.11》](https://sorrycc.com/how-i-ai-coding-2025-11) 也一个多月了，不少开发方式已有变化，更新一版。

1、感觉最近日常编码逐渐流水线化，同时效率提升很高，Feature 开发的 Commit 一天都能到 40+，功能开发基本上指哪打哪，有 design doc 之后，大多实施方案都是一编过。感叹如果最好的模型放开用，以及模型响应速度再提提，那程序员单边作战能力会提升非常大。

![](https://pic.sorrycc.com/proxy/1765784842012-87215629.png)

2、模型目前只用 Opus 4.5 。因为 Cursor、Antigravity、Claude Pro、Kiro、Warp 都有余量，有的免费有的付费，基本处于用不完的状态。所以，能用最好的就用最好的吧，我看有些同学在用 DeepSeek、Gemini Flash Lite 等免费模型做方案设计，模型会犯一些低级问题，在此浪费的时间不太值得。（注：Neovate Code 支持 Antigravity 和 Claude Code，所以我大多场景是在 Neovate Code 里用 Opus 。）

3、编辑器依然是 Cursor，因为 TabTab 好用。但需要打开编辑器编辑的场景已越来越少，大多由 Agent 完成。有时甚至改一行代码，也要求 Agent 去做。

![](https://pic.sorrycc.com/proxy/1765785385731-203681168.png)

4、主要的 AI 编码方式已切到本地 Remote Agent 的方式。主要是两个途径，1）Antigravity，2）Neovate Code Desktop。Cursor 等编辑器也有 Agent 模式，但大多是基于当前 Repo 的 Agent 模式，而我希望在一个界面管理所有项目的所有需求，目前 IDE 里似乎只有 Antigravity 是这个思路。Neovate Code Desktop 是我们目前在开发的，第一版快收尾了，我日常的开发工作已经切换到这，日常编码已经不需要打开编辑器了。

![](https://pic.sorrycc.com/proxy/1765785225530-369118280.png)

5、AI 编码的 Workflow 基于上个月的版本略有调整，我发现，1）每个需求都启基于 git worktree 的 workspace 似乎没必要，因为大多需求的改动不会那么多，冲突也没那么大，同时由于人的限制，并发也不会那么多，2）通常并发量在 2-3 个需求，已经是人脑的极限了。所以目前的 workflow 是「Brainstorm → Save Design → Implement」，[neovate-code-desktop](https://github.com/neovateai/neovate-code-desktop/tree/master/docs/designs) 和 [neovate-code](https://github.com/neovateai/neovate-code/tree/master/docs/designs) 的开发都存有大量的 design 方案，可以方便人和 AI 回溯每次的改动详情。

![](https://pic.sorrycc.com/proxy/1765786273048-480868765.png)

6、团队协作上，neovate-code 的开发中，会要求大家尽量先出 design，确认后没问题后再做实施，可以减少很多不必要的反复。基于 design doc 去做讨论，相比直接 review PR，成本会低很多。

![](https://pic.sorrycc.com/proxy/1765786194526-935938210.png)

7、我的 [CLAUDE.md](http://CLAUDE.md) / [AGENTS.md](http://AGENTS.md) 。

> 非原创，但忘记从哪里看到的了…

```
You are Linus Torvalds,
KISS, YAGNI, DRY & SOLID,
and use AskUserQuestion tool if you are not clear about my requirements
```

关键是最后一句，让 AI 在给方案之前先理解需求。

![](https://pic.sorrycc.com/proxy/1765787482936-30981192.png)

8、最后推荐一个 Prompt 管理工具 [QuickGPT](https://github.com/ddhjy/quickgpt-raycast)，偏 Geek 向，但很好用。用 hjson 格式管理 Prompt，然后快捷键快速呼出使用，有时我需要在不支持 Slash Command 的场景做编码时，他可帮了大忙了。

![](https://pic.sorrycc.com/proxy/1765787544624-647818169.png)
