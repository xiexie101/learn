# 228 - 《5 种 ChatGPT 的扩展方法》

ChatGPT 太流行，一周时间各种扩展都有了，Chrome 插件、VSCode 插件、微信小程序、Telegram 机器人、Twitter 机器人等。ChatGPT 并没有提供官方 API，那他们都是通过什么方法实现扩展的？目前看到的有 5 种。

1、用的最广的是 [chatgpt npm 包](https://github.com/transitive-bullshit/chatgpt-api)，基于反向分析实现的 node api，80% 的机器人和插件应该都基于此。昨天发布的 2.0 版本支持了 conversation，功能上已经完整了。简单翻了下源码，发现 chatgpt 的 Server 是 SSE（Server-Sent Events）实现的，所以客户端实现时借助了 [eventsource-parser](https://github.com/rexxars/eventsource-parser)，等所有 chunk 全部返回才最终给到响应。鉴权需要自己取 cookie 传入。

2、Chrome 插件通常是在方案 1 的基础上，少了鉴权环节，只要自行去 [chat.openai.com](http://chat.openai.com) 登录即可，Chrome 插件再请求 API 时会自动带上 cookie。比如 [chat-gpt-google-extension](https://github.com/wong2/chat-gpt-google-extension)。

3、基于 playwright 的简单封装，可以本地启 server 启动 api，比如 [chatgpt-api](https://github.com/ZohaibAhmed/chatgpt-api)，利用 headless 浏览器模拟点击和跳转。鉴权是自己在弹出的浏览器里输入账号密码。[chatGPT-telegram-bot](https://github.com/altryne/chatGPT-telegram-bot) 也是这种实现。

4、直接套个 webview 指向官网的，适用于原生应用，比如 [chatgpt-mac](https://github.com/vincelwt/chatgpt-mac/blob/main/index.html#L10)。

5、增强输入框类。比如 [chatgpt-advanced](https://github.com/qunash/chatgpt-advanced) 这个 Chrome 插件，会拦截用户输入，比如用户输入「how old is TangWei」，他会请求 [https://ddg-webapp-aagd.vercel.app/search](https://ddg-webapp-aagd.vercel.app/search) 去拿搜索数据，然后把在搜索数据的基础上，加上「Given these web results, answer the following question: how old is TangWei」后提交给 chatgpt，让 chatgpt 具备网络能力。
