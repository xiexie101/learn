# 431 - 《我读文章的流程》

![](https://img.alicdn.com/imgextra/i1/O1CN01L3pXPB1ji9KMdCxLm_!!6000000004581-0-tps-1920-1079.jpg_1200x1200.jpg)

题图：esmiloenak @ unsplash.com

> 上周群里有人问，我再整理了下。

之前在 [MDH Weekly 124 - 《Code Like a Pro》](https://sorrycc.com/mdh-124) 写过一段。

> 上周更新了不少译文，原因是我手写了两个脚本来改进翻译流程，效率高了很多。现在的流程是，1）先用 bookmarklet 把网页转成 markdown 并保存，2）然后用脚本翻译 markdown 为中文，3）阅读，并在阅读时做高亮和笔记，4）加 frontmatter 属性，用脚本同步到 [sorrycc.com](http://sorrycc.com) 。
> 
> 介绍下两个脚本。1）一个是网页转 Markdown 的脚本，其中识别代码部分花了不少时间，因为各个网站用的语法高亮器不同，要从中找出代码和语言是个体力活。2）另一个是针对 Markdown 的翻译脚本，相比之前用 Immersive Translate 的好处是，a）会有更好的上下文，因为会把多行合到一起翻译，而不是每一行单独翻译，b）代码里的注释也能翻译，c）链接啥的全保留。

我的基本流程是：

1）先用 bookmarklet 把网页转成 markdown 并保存  
2）然后用脚本翻译 markdown 为中文  
3）阅读，并在阅读时做高亮和加「编者注」  
4）加 frontmatter 属性，用脚本同步到 [sorrycc.com](http://sorrycc.com)

阅读时做高亮和加编者注可以让我后期看到这篇文章时减少陌生感，不用重新再读一遍；而同步到 [sorrycc.com](http://sorrycc.com) 满足分享需求，让更多对此感兴趣的同学收益。

要实现这个流程，需要围绕 Markdown 文本的两个脚本。一个把 html 转 markdown，一个基于 markdown 做翻译。第一个脚本上周五直播时[开源](https://github.com/sorrycc/markify)了，ChatGPT 取的名，叫 markify，目前已可用，但还有 PR 在重构中；第二个脚本我尽量这周也开源下（开源后更新本文）。

Why markify？最早我是在用 [MarkDownload](https://chromewebstore.google.com/detail/markdownload-markdown-web/pcmpcfapbekmbjjkdalcgopdkipoggdi) 这款 Chrome 插件的，但是发现他对于网页里的代码块处理不好，要么无法识别到语言，要么代码换行丢失，导致需要额外做不少手工操作。于是就自己写了，基于 [Turndown](https://github.com/mixmark-io/turndown) 做 Html 到 Markdown 的转换，然后针对不同站点和高亮工具做代码块的适配。

Why 基于 Markdown 做翻译？之前我是用的 Immersive Translate 在网页侧做翻译，然后再转 Markdown 保存。但发现有几个问题，1）上下文丢失，Immersive Translate 是逐行翻译，这肯定没有把多行放一起的效果好，2）链接丢失，导致后期不少手动补链接操作，3）代码里的注释没有翻译。而自研 Markdown 翻译可以解上述问题。
