# 515 - 《我的信息流 2025.1》

AI 时代了，上一次写信息流已经是 2023 年了，见 [241 - 《我的信息流 2023.1》](https://sorrycc.com/information-flow-2023)。借助 AI 的能力做下调整，让信息获取和输出更有效率和质量。

1、理论。

我把信息分为三类，每一类有不同的处理方式。

1）咨询，看过标题就可以直接过了  
2）泛读，看个大概，这类的看 AI 总结就够了  
3）精读，可能是专业相关，或者文章写的很好的，需要保存、翻译、高亮、总结等

我记性一般，看过之后过段时间会忘，所以得留点痕，读过的要有记录，以方便我后续通过搜索等方式找到。

AI 时代，还守着老方法就比较低效了。借助 AI 做信息获取、总结、阅读，可以快速过一篇文章或一本书，然后再挑重要的精读，这会比之前更高效一些。另外，使用 AI 得权衡，我在用的过程中，隐隐有点担心的是，过度利用 AI 可能会让自己减弱思考和动手的能力。

2、实践。

我有 3 个文件夹。

```
+ ArticlesInbox
	+ 202401
	- Artitles_202401.md
+ ArticlesPendingTranslations
+ ArticlesTranslated
```

1）ArticlesInbox 是收集箱。按月保存，同时附带一个索引页。添加基于脚本。

```bash
# 添加文章信息（title 和 summary 不指定会自动获取）
$ ,s article --url URL --title TITLE --summary SUMMARY --tag A,B,C
# 交互式
$ ,s article
```

他会把文章保存到 ArticlesInbox 目录。同时添加索引信息到 Articles\_YYYYMM.md，索引文件的格式如下。

```
## 2025.01.23

- **[TITLE](URL)**，SUMMARY。`#tag1` `#tag2` [[ARTICLE in LOCAL]]
- **[TITLE](URL)**，SUMMARY。`#tag1` `#tag2`
```

![](https://tcc.sorrycc.com/p/27062fc4-0a30-4199-93e9-b9c7cf3ed7d4.gif)

注：精读类型加 `--save` 参数，会额外保存原文，并加上链接，方便后续翻译、搜索和精读。保存 html 为 markdown 有两种方式，1）用微软的 [markitdown](https://github.com/microsoft/markitdown) 把 Html 转成 Markdown，缺点是代码提取不太准，所以不适用于包含代码的文章，2）用 [markify](https://github.com/sorrycc/markify)，优点是对代码提取友好。

2）如果一篇文章需要翻译，我会在页头加上 `needTranslate: true` 的 yaml 信息。然后执行脚本。

```bash
$ ,s move
```

脚本会找到所有需要翻译的文章， 迁移到 ArticlesPendingTranslations 目录。

![](https://tcc.sorrycc.com/p/9e34d44c-4eb6-4cd3-9b22-717fd69b8028.gif)

再执行翻译脚本。其会找到所有 ArticlesPendingTranslations 目录下的 markdown 文件，翻译、备份原文、生成译文到 ArticlesTranslated 目录、加上对应的 yaml 信息头、删除 `needTranslate: true` yaml 信息、同步信息到 Index 页。

```bash
$ ,s translate
```

![](https://tcc.sorrycc.com/p/518dea31-1c67-4235-899b-964f0aa4567e.gif)

3）如果一篇文章需要发布到 [sorrycc.com](http://sorrycc.com)、telegram、twitter 等平台，我会在文件头部加上 `needPublish: blog | telegram | twitter` 信息（注：目前只支持了 wordpress 平台的 blog）。然后执行脚本。

```bash
$ ,s wordpress
```

相应文件会被发布，并同步信息到 Index 页。

![](https://tcc.sorrycc.com/p/ad682d56-a8a1-4ae3-b009-1cde28f7842f.gif)

注：不太长的文章，会更倾用 [508 - 《用 Cursor 半自动地整理资讯》](https://sorrycc.com/use-cursor-to-semi-automate-news-collection) 的方法，在 Cursor 里白嫖 Claude-3-5-sonnet 处理。
