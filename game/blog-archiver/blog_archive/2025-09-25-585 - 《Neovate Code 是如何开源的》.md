# 585 - 《Neovate Code 是如何开源的》

[Neovate Code](https://github.com/neovateai/neovate-code) 昨天开源了，继续记录下大概的流程。

1、这次开源和之前的 umi、dva、mako 等不同，流程复杂点。之前是就直接就开了，也没有走什么流程，因为 umijs 这个 org 申请过，所以放下面就不需要重复申请。而这一次我们走了完整的品牌申请和开源申请的流程，时间跨度比较长。品牌申请需要确认国内和国外是否已有冲突，这个细节比较多，就不展开了。

2、完成流程之后，也还做了和开源相关的不少功能补全。比如各种 provider 以及和 provider 相关的 /login 和 /logout 功能，这在内网是用不到的，内网有内置的模型，不需要用户注册。同时也还有做 bench，用黑盒的方式来验证 Code Agent 的效果如何。以及还有做 windows 兼容和无 node 环境的 install 脚本等，都是为了让新人更容易上手。

3、接下来就是官网、文档和发布稿了。

官网和文档用的 next.js + nextra，搭完之后有同学说 nextra 过时了，现在流行 [fumadocs](https://fumadocs.dev/)，可惜完了点，时间不够了，不然我就换了。。同时文档架子是让 ai 写的，写完后猜发现他用了老版本，next.js 4 和 nextra 3。。

nextra 搭文档相对还是比较省心的，功能比较全和稳。文档我先写了英文版，老板说必须得有中文，于是我让 ai 翻译了一版，基本没动，大家如果读文档，应该是能感受到淡淡的 AI 味的。

发布稿也是先用英文写，然后让 ai 翻译成中文，但 ai 味太浓，于是手动逐句调整了一遍去 ai 味。之所有文档和发布稿先用英文写，倒也不是因为我的英文更好，而是 ai 辅助补全英文的内容更顺滑。对了，我是在 cursor 里写的文档。

发布稿的架子是用的外滩大会分享的 PPT 的结构。

4、Delay 了 5 天。

本计划是 9.19 发布，但是发现文档、官网和发布稿比想象中需要更多时间，周末又不想加班，于是 Delay 到 9.24 。正好是我小儿子的生日，也算有了一层额外的意义。

5、最后是运营。

没啥运营经费和团队，买不了广告位和推广，所以主要还是通过自己和团队的公众号、群转发等途经。话说某公众号一个专栏得 40W 。。通知精力比较有限，目前只做了国内的运营，x 和 youtube 还没有去。同时由于 Neovate Code 是 BROK 的方式，你需要用自己的 API Key。所以也没有办法通过「白嫖」的方式吸引到用户。

有一点和之前不同的是群二维码用了活码，这样群满就可以换个群继续加了。方法是 [https://cli.im/](https://cli.im/) 上用图片生成二维码，然后不断去更新那张图片即可。

发布当天 300 Star 其实不算多。
