# 531 - 《AI x 程序员 x 蚂蚁》

> 内部分享和讨论版，不是终态版本。

1、基于 [530 - 《我怎么用 AI 辅助编程》](https://sorrycc.com/how-to-use-ai-to-assist-programming)，我觉得未来的 AI 开发方式 = LLM + Planer（结合框架 tools、最佳实践、标杆项目等，产出架构文档和实施方案）+ Code Agent（修改代码、执行任务）。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/qwNvvN.png)

需求经过 PLAN AGENT，产出 Step By Step 的实施文档，然后再经过 ACT AGENT 基于实施文档修改代码和执行 Tools。其中 PLAN AGENT 和 ACT AGENT 都会调用 Tools、最佳实践、标杆项目、项目描述、现有代码等。

所以，未来开发，搞个标杆项目，[repomix](https://repomix.com/) 一下，然后让 AI 在新项目里照着写就好。

2、现在社区基于 Cursor、Claude-3-5-sonnet、Gemini 2 的开发流程已经很流畅了，保守评估的话，提效 2 倍也肯定是有的。对于蚂蚁内部来说，处于信息安全的考虑，和外部主要是模型的差异，现在最先进可以用的是 DeepSeek V3 和 R1。但模型的迭代很快，感觉很快就能用上更智能量更大的。所以，我们可以先把工具准备好，提前布局工具和基础设施。

3、社区「AI x 程序员」的有几个大的方向。

1）类 Bolt.new、V0.Dev、Replit 的研发平台  
2）类 Cursor、WindSurf、TRAE、[AIDE](https://github.com/codestoryai/aide) 的编辑器  
3）类 Github Copilot、Cline、RooCode、[CoolCline](https://github.com/coolcline/CoolCline)、通义灵码、[Cody](https://github.com/sourcegraph/cody) 的编辑器插件  
4）类 Aider、Claude Code、Gemini Code Assit 的命令行 Code Agent  
5）开发 Tools（MCP Server）集合，方便前面 2-4 的工具调用

4、一个思路是做 Code Agents（包括 PLAN 和 ACT）， 3 和 4 可以一起做，两者不冲突，3 基于 4，是在 4 的基础上增加了 GUI 的交互逻辑、编辑器的环境信息（比如当前打开的 Tab、编辑的文件、选中的行、Terminal 运行结果、Lint 出的 Problems 等）。

5、如果要做 3。可行的一条路是基于开源的 Cline 或者他的 Fork 版本 Roo-Code 来做（注：纯 Code Agent，没有代码补全），我也有写过一个 Toy Cline 来验证可行性。接入蚂蚁要做的是，在 Cline 基础上，增加对内部平台、框架、文档、服务、最佳实践的适配。这个适配可以通过添加文档引入更多上下文，或者通过 Tools（MCP Server）的方式进行扩展。

6、不过由于内部已经有 NeoCoder、WeaveFox、CodeFuse（含 SVAT）等 AI 产品，以及最近出的 Claude Code 的体验也挺好。我倾向先从 4 做起，用命令行实现 Plan Agent 和 Code Agent，可以快速实现 POC，等把基础打实了后，再通过 VSCode Plugin 做更好的交互，这一层可以和其他团队合作。
