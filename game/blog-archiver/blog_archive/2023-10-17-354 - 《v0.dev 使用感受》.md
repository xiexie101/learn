# 354 - 《v0.dev 使用感受》

> 试验了下 v0.dev，做了个博客首页和文章页，200 credit 很快就用完了。见 [https://v0.dev/r/8ACvrqoqXgE](https://v0.dev/r/8ACvrqoqXgE) 和 [https://v0.dev/r/b9vkM5D30nb](https://v0.dev/r/b9vkM5D30nb) 。

1、价格不贵，但可以更低。价格看和什么比了，和 GPT-4 的 API 比肯定是贵的。算了下，最便宜的套餐 $10/M 给 1500 个 credits，能跑 150 次 prompt，平均 5 毛钱人民币一次，而 GPT-4 API 远不到这个价格。但如果和设计 + 前端的人工价格相比，简直便宜到离谱。

2、不过我倒觉得 v0.dev 可以卖更低。因为大家在上面创造的内容才是更大的价值，如果形成社区，程序员都用这个来做网站设计和寻找灵感，那感觉能做更多是。比如卖模版，和作者分成等等。同时技术很容易被模仿，而社区很难被复制。除非 vercel 本身也觉得 v0.dev 的需求并不大，大家都是用完即走的使用方式。

3、一个省 credit 的 Tip 送给大家，就是可以一次性把需求描述清楚，而不是像我一样边生成边调整需求。

4、体验上整体挺顺的。在 v0.dev 上开发后，可以通过 npx v0 add xxx 快速集成到本地开了 TypeScript 的 Next.js 项目中（注：目前只支持 Next.js 项目）。初始 prompt 会并行跑三份，给 3 个可选。然后可基于此做迭代，他会基于你给的 prompt 在原网站的基础上调整代码和设计。你还也可以通过 Inspect 工具选中页面的部分区域并只调整这部分界面，见下图。

![](https://img.alicdn.com/imgextra/i2/O1CN010ZWYuN1DDOyKvD60t_!!6000000000182-2-tps-1520-928.png)

5、产物是 Tailwind CSS + Shadcn UI，同时考虑的响应式，代码比较简洁，但不包含 Shadcn UI 的代码，这部分应该是假设已经存在于项目的 components 目录的。产物里生成的 ICON 效果还不错，感觉这部分应该是花了不少精力去调整的。动画部分我没有尝试，不清楚是否支持。

6、v0.dev 适合于谁？感觉有几个场景，比如，1）设计感不强的程序员，比如我，因为我觉得他产出网站的色系和风格都还挺好看的，可能比网上的大部分网站都还要好一些，2）业余项目，3）项目的初始化阶段，4）后端程序员。

7、缺点？待补充。

8、还有个脑洞是，淘宝/抖音开店接单，直接用 v0.dev 生成交付。如果客户不满意，直接把客户的意见贴进去让 AI 迭代就好，每次 1 块钱，还能赚 5 毛，哈哈。
