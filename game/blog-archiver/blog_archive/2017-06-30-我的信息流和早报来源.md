# 我的信息流和早报来源

![](https://img.alicdn.com/imgextra/i4/O1CN01pYPZny1ZEhfWPI1me_!!6000000003163-0-tps-1920-1280.jpg_1200x1200.jpg)

题图：markusspiske @ unsplash.com

因为经常发早报，所以不少同学会对我早报的信息来源感兴趣，这里做下整理。

## 信息流

![](https://img.alicdn.com/imgextra/i3/O1CN01IdXBoj1yKX6XWs1uh_!!6000000006560-2-tps-2012-1098.png)

我的信息流基本分了几个阶段，

1.  先从各种渠道到 RSS
2.  RSS 和其他渠道到 Instapaper
3.  Instapaper 到 sorrycc/readings（私库）（自动）
4.  sorrycc/readings 的 issue 打 tag 后做各种用途，比如早报、Blog、Vlog 等

### 各种渠道到 RSS

RSS 现在用的人相对少了，但作为信息源增量更新的一种方式，我觉得对我来说很适用，所以会尽量把一切信息源转为 RSS，然后通过 reeder 进行订阅。

看到一个不错的信息源，我的步骤是，

1.  先看他自身有没有 RSS 服务
2.  去 [rsshub.app](https://docs.rsshub.app/) 和 [rss.lilydjwg.me](https://rss.lilydjwg.me/) 找有没有封装
3.  在 [feed43.com](https://feed43.com/) 基于正则转一个出来
4.  自己实现一个 serverless 服务，部署在 [zeit now](https://zeit.co/now) 上

3、4 是并列关系，以前用 3 比较多，现在用 4 比较多。

然后，还会有不少渠道是 RSS 不能覆盖的，比如微信、twitter 上随意看到的、朋友或朋友圈或群里看到的链接，等等。

### RSS 及其他渠道到 Instapaper

RSS 过来的是海量的信息源，经过筛选之后的部分精品文章我会让他留到 Instapaper。另外，微信、浏览器访问时看到的文章也会添加到 Instapaper。

至于为啥用 Instapaper？

1.  主要是添加文章方便，渠道很全，基本上不用担心在某些场景下不能添加的问题
2.  老牌服务商，稳定

我可以在 Chrome，Reeder（mac 和 iPhone），微信，twitter 等地方，一键保存文章到 Instapaper。

但 Instapaper 的角色并不是 to read later。

注：

1.  iOS 下的 Instapaper 没有在过去上架，需切到美区才能安装
2.  微信到 Instapaper 需借助 Workflow/Shortcuts，参考知乎的[这篇回答](https://www.zhihu.com/question/23448653/answer/151331421)

### Instapaper 的角色

Instapaper 其实是个中转站，除了手机上，我很少会在他上面看文章，而是通过他自动归档到 [sorrycc/reading](https://github.com/sorrycc/reading)。

### Instapaper 到 sorrycc/reading

具体可以看 [Serverless 实战：打造个人阅读追踪系统](https://blog.jimmylv.info/2017-06-30-serverless-in-action-build-personal-reading-statistics-system/)，主要是通过 ifttt，如果有 Instapaper 有新的 item，在 github 上创建一个 issue。至于在 Instapaper 里存档后自动关闭 github 上的 issue，我也做了，但实际下来感觉用处并不大。

为啥要归档？

*   存在 github 上比较放心
*   github issue 可以添加笔记、tag、archive 等

### sorrycc/readings 到输出

输出可以有多重类型：

1.  早报输出
2.  脑图用 iThoughtsX 或 Bear
3.  简短笔记用 Bear
4.  长篇用 Typora

比如早报输出，我有写一个脚本，可以收集 issue 里打上早报 tag 的新 issue，并且能智能地区分`发布`和`类库`的分类，这样我前几天看的内容就能一键生成早报。

比如我要学 React Hooks，在看完官方文档后，我会在 sorrycc/reading 里搜 react hook 看哪些之前存的精品文章。

## 早报来源

### twitter

#### List

我有整理一个 list [https://twitter.com/chenchengpro/lists/fed](https://twitter.com/chenchengpro/lists/fed)，上面是我关注的一些前端大牛，信息较多，我只看 star 50 以上的。

#### Front-End Programming Trend

twitter 里有个 front-end programming 的 trend，入口很隐蔽，很多人不知道的。在 [https://mobile.twitter.com/](https://mobile.twitter.com/)，点上面的井号，然后下拉，如果你有关注前端相关的人，就会有 Front-End Programming 的 trend，见下图，

![](https://img.alicdn.com/imgextra/i3/O1CN01IbdplF1HhrnzxatPu_!!6000000000790-2-tps-2570-1204.png)

### github

#### 类库发布

先在类库的 github 上点 Watch -> Release Only，这样在他们有发布版本时会收到消息，我自己是 watch 了 [awesome-f2e-libs](https://github.com/sorrycc/awesome-f2e-libs) 里所有的类库，然后接收是在 [https://github.com/notifications](https://github.com/notifications) 。

#### follow 大牛

follow 后会有个私有的 rss 源（叫 Github Private Feed，但具体在哪一时间找不到了。。），包含你 follow 人的所有活动，star、create、fork 等，然后你在 reeder 里订阅他就好。  
但是有个问题，这里有不少冗余信息，比如很多人 follow 了我的库也会出现在这里。所以我是写了个 serverless 服务做过滤的，地址在 [https://feed.sorrycc.com/github](https://feed.sorrycc.com/github) 。

#### 自己的 star

自己在之前 star 的仓库，比如[我的](https://github.com/sorrycc?tab=stars)，在整理早报时也会通过脚本一键输出更新的内容。
