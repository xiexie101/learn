# 72 - 《我司前端那些事儿》

时间仓促，简单梳理了下。

1、蚂蚁的前端体系从开源社区应该就能感知到很多。我们有开源的传统，从最早的 seajs、aralejs、spmjs，到现在的 antd、antd-pro、antv、umi、dva、egg、tnpm、qiankun、ahooks 等等。不少同学都喜欢直接开源做方案，包括我。这样可以收到来自社区的大量反馈，至少不会坐井观天。

2、内部需求和社区需求是有差异的，所以基于开源的 umi 和 egg，内部会有对应的 bigfish（中后台） 和 chair（node） 框架。除了 bigfish，我们还有服务 h5 的 smallfish 和服务小程序的 minifish。三条🐟构成了蚂蚁的前端框架（其实我觉得两条就够了）。基于三条🐟，业务项目允许适当发挥，但不允许低水平重复造轮子。

3、内部在 antd 和 antd-pro 的基础上，也有扩展业务组件库 techui，部分组件使用率还不错。所有 UI 库都会上传到统一的资产平台进行管理。

4、蚂蚁 React 和 Vue 的占比大概是 95% vs. 5%。

5、我们有大量的自研服务和平台。比如自研的 devopts 平台、自研的低代码平台、自研的资产平台、自研的 git 服务（再之前是用 gitlab）、自研的 CloudIDE 平台、自研的前端数据服务（OneAPI）、微前端服务、测试平台、unpkg 服务、代码片段和 DEMO 平台、CI 服务、数据服务等等。只要能想到的有点价值的，内网都有竞品。

6、低代码平台在内部是强推的，项目支持策略是低代码 First。面向的用户群变化了好多次，目前主要面向非专业前端，比如全栈、外包等。利益相关，不方便评价好坏。

7、Bundless CDN 目前比 Bundle 慢，还在优化中，暂未推行。也有可能是 Bundle 方案优化地太好了。。
