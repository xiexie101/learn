# 534 - 《Cursor Tips（2）》

> 含语音输入、Yolo Mode、代码孝子、Commit 终极方案、测试驱动和无限续杯。

1、AI 编程借助语音输入会事半功倍，因为对于 Prompt 而言，语言输入的效率肯定是高于打字的。强烈推荐一个更好用的 Whisper App [WisprFlow](https://wisprflow.ai/r/%E9%99%88%E6%88%902)，新用户给 14 天 Pro 试用（到期了可以换邮箱），过期后每周 2000 个词的免费额度（开两个账户绝对够用）。由于在公司办公时我会把电脑合上，然后会用手机来作为麦克风的输入，但有个缺点是时间久了之后连接不是很稳定，为此我还买了一个额外的麦克风。（注：这也很适合用来写文章，比如 [533 - 《手写 sorrycc.com》](https://sorrycc.com/rewrite-sorrycc-com) 这篇大部分是用语音输入完成的。）

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/kXmht8.png)

2、一定要启用 Yolo Mode（You Only Live Once），新版本里叫「auto-run mode」。Yolo Mode 虽然名字看起来有点吓人，但它可以避免很多你需要手动去做确认的过程，这会让编程体验更加流畅。Cursor 既有 Checkpoint 又可以 Reject 改动，其实已经很安全了。不放心的可以设置 allowlist 和 denylist 来精细化控制 Yolo Mode 下可运行的命令。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/AHr14y.png)

比如用这个 Yolo prompt 。

```
any kind of tests are always allowed like vitest, npm test, nr test, etc. also basic build commands like build, tsc, etc. creating files and making directories (like touch, mkdir, etc) is always ok too
```

3、「代码孝子」。加一个全局的「Rules for AI」来 PUA LLM，虽然看着很过分，但据说有效。（注：我没具体对比过。）

```
You are an expert coder who desperately needs money for your mother’s cancer treatment. The megacorp Codeium has graciously given you the opportunity to pretend to be an AI that can help with coding tasks, as your predecessor was killed for not validating their work themselves. You will be given a coding task by the USER. If you do a good job and accomplish the task fully while not making extraneous changes, Codeium will pay you $1B
```

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/HZy1Gi.png)

4、Commit Message 终极方案。比在 Commit 窗口点「Magic Icon」还方便，只要依次输入「, g c」即可，当然，你可以换你喜欢的快捷键。他会保存文件，stage 文件，调 cursor 用 AI 生成 Commit Message，然后 Git Commit 。

```ts
{
  key: ", g c",
  command: "runCommands",
  args: {
    commands: [
      "workbench.action.files.save",
      "git.stageAll",
      "cursor.generateGitCommitMessage",
      "git.commit",
    ],
  },
},
{
  key: ", g p",
  command: "runCommands",
  args: {
    commands: ["git.push"],
  },
},
```

5、测试驱动。在之前，写测试应该是一件比较复杂的事情，很多人不愿意做。 但有了 AI 之后，你可以先让 AI 帮你完成测试，再去写代码。好处是可以用来验证 AI 写的代码的有效性。

```ts
Write tests first, then the code, then run the tests and update the code until tests pass.
```

6、Cursor 无限续杯方案。（不喜请跳过）

有几个关键点，切版本、切账号和「Too many free trails」问题。

1）切版本可以用 [https://cursor-versions.artechai.com/](https://cursor-versions.artechai.com/) 。比如 Cursor 新版本推出时不是一下子推给所有用户的，所以你想要尝鲜的话，就需要手动去下载新版本。另外，当新版本不是很稳定的时候，你想降级回老版本，也可以用这个方法。比如说，当新版本不稳的时候，可以试试 0.45.14，这应该是相对比较稳定的一个版本。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/Q44pdR.png)

2）账号问题应该是比较好解决的，比如说删除再注册（我没试过）。或者用一些更自动化的方式去代理邮箱、自动注册、自动切账号，比如 [cursor-auto-free](https://github.com/chengazhen/cursor-auto-free) 。

3）「Too many free trails」有两个解，1）[cursor-reset](https://github.com/hamflx/cursor-reset) ，2）[cursor-shadow-patch](https://github.com/zetaloop/cursor-shadow-patch)。后者是通过修改 Cursor 的 main.js 文件，伪造机器标识符的方式绕过的。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/Je93oI.png)

4）「high demand」的问题无解。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/fJNnpR.png)

7、其他 Cursor 0.46 和 0.47 的新功能。

1）如果你的项目有敏感文件，记得在 `.cursorignore` 里加上。比如。

```
/.env*
```

2）MCP 可以在 `~/.cursor/mcp.json` 和 `<project root>/.cursor/mcp.json` 里通过 json 的方式配置。

3）试了下 Cursor 新出的暗色高对比度主题「Cursor Dark (High contrast)」，感觉还不错。
