# 523 - 《Cursor Tips（1）》

> 9 个之前不会的 Tips 。

1、从 package.json 提取依赖并添加到 cursor rules 中，这样 Composer 就能知道有哪些可用的包。让 Cursor 更了解你的项目。

```
@.cursorrules @package.json Extract all of the dependencies from my package.json file, add them to my allowed packages in my cursor rules file, and give a brief explanation of when and why to use each package.
```

2、搜索结果页有个正方体 ICON，点击后会把所有结果都放到 Cursor Composer 里，方便让 AI 批量修复代码问题。

![](https://tcc.sorrycc.com/p/997fc3f0-8b93-4bb9-8d32-115d9cbbb5ed.png)

3、你可以把链接直接发给 Cursor，比如 eslint rule 的链接，他会读链接内容的，然后基于链接内容做修改。

4、Composer 里使用 Agent + Yolo Mode 体验更流畅，Yolo 即 You Only Live Once，执行命令时不会和你确认。

```bash
download @https://makojs.dev/llms.txt  to the tmp folder
```

5、可以用 gh 搜索代码，然后拉代码到本地，交给 AI 分析。也可以试试 [ghx](https://github.com/johnlindquist/ghx)，gh 搜索代码的进阶版。

```bash
gh search code "react useEffect language:typescript" --json url
```

6、可以用 [Superwhisper](https://superwhisper.com/) 做语音输入，在你方便的时候，通常语言表达会比打字更快一些，进一步提效。因为感觉思维到语音相比思维到打字的转换，损耗更少一些。更多 Whisper 工具见 [awesome-whisper](https://github.com/sindresorhus/awesome-whisper) 。

![](https://tcc.sorrycc.com/p/ad886ce5-da7d-465c-b875-483dd5fe6936.png)

7、在 Compoer 窗口按 「cmd+.」可以切换 normal 和 agent 模式。

8、对于复杂任务，用强大的 Reasoning Model 比如 o1 pro 或 deepseek r1 先拆任务指定详细计划，保存计划到文件，再让 Composer 分步执行任务，也是个不错的方法。

```
@plan.txt Work on step 1
```

9、Agent 可以结合 Git Hook 使用。在 Git Hook 里做 Prettier、Lint 之类的事，然后在 Composer（切 Agent 模式）里做 commit，Cursor 会自动帮你修复问题。

```
Let's check this in!
git add.
git commit -m "add a good message here"
git push
```

10、先用 ghx 从 github 找示例代码，然后再让 Cursor Agent 修复问题，会更准确。

```
npx -y @johnlindquist/ghx "@modelcontextprotocol/sdk/server/mcp.js" "SSEServerTransport" --context 200
```
