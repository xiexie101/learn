# 258 - 《读书笔记：营销笔记》

> 昨天翻完的，趁着热乎，记下笔记。

1、做营销注意三件事。1）做事要基于能力和资源，不能基于美好的愿望，比如老鹰可以吃到兔子，鲨鱼却吃不到，但鲨鱼不应该羡慕老鹰，而是应该努力捕捉更多的鱼；2）要基于用户的视角和偏好，不能基于自己的，比如跳广场舞的大妈，你可能觉得她的服装品位很土，但她一定觉得自己穿的是最好看的衣服，3）基于用户的需求和利益去思考和行动，不能基于自己的，比如写软文的自媒体纯粹是为了自己的收入而忽略了用户利益。

2、企业成功是因为具备某种竞争优势，而竞争优势的获得必须符合经营逻辑。比如茶企业想增加业务，做茶饮料就不符合经营逻辑，因为赛道完全不同，而开发喝茶的器具就符合经营逻辑，因为茶具是可以卖给相同的消费者的。比如程序员如果要搞副业，也要充分利用、发挥、积累和放大自身的优势，比如编程相关的专栏、小册、课程、书，这样成功率会更大。低价是优势吗？不是，低成本带来的低价才是优势。

3、所谓巨人过河，是不需要策略和技巧的，巨人直接就走过去了，只有小松鼠过河才需要策略和技巧。（还有句类似的话叫，只有弱者才需要讲道理，强者就不需要这个技能的。）比如戴森吸尘器在产品创新上有碾压式的优势，不用多余宣传，只要把产品摆在那让顾客体验一下就可以了。

4、企业要取得卓越有两种做法，提升经营效率和进行战略定位。1）日本企业善于提效，对每个环节进行完善，不断提升自身效率，就能降低成本，2）战略定位，我理解是换了个赛道，是一种降维打击的方式，比如前端太卷，那么你可以选择改用 Rust 写工具。

5、营销是什么？科特勒给的定义是：企业为从顾客处获得利益回报而为顾客创造价值并与之建立稳固关系的过程。解释下，1）创造价值，产品或服务必须是对用户有价值的，2）获得利益回报，企业不是慈善机构，3）建立稳固关系，只要不是一次性交互的关系，就需要考虑如何维护和锁住老用户的问题。比如这个星球的价值是什么，我觉得是通过写日更给大家带来视野层的一些启发，如果出现问题，要从价值开始排查问题，看是否是提供的价值消失或降低了。

6、市场营销有个 4P 框架，包括 product（产品）、price（定价）、place（渠道）、promotion（促销）。4P 框架牛逼的地方在于是恒定的、不会过时的，不管技术怎么发展，不管渠道如何转变，不管购买行为如何变化，这个框架都极为稳定。前端变化这么快，他的恒定的 4P 是啥？我觉得可能是「URL > 数据 > UI 渲染 > 交互」这个循环（注意：交互后走回数据）。

7、有个概念叫 ROI（投资回报率）。比如线上广告投放，收入除以费用就是 ROI，ROI 为 1 就不赚钱了。我之前尝试在其他公众号投了两次这个星期的广告，ROI 就是 1，有时甚至就 0.x，就没有继续了。但是，还得看一个数据叫「复购率」，如果这个数据高，也是划算的。

8、烂市场里做第一，也不如好市场里做第二十。比如选种子，如果选红杉种子，就能长成参天大树；如果选马尾草种子，长大之后最多就是一棵比较高的马尾草而已。这让我想到前端的公众号，前端领域的市场比较小，做到第一的阅读数可能还不如后端领域的第二十。

9、产品越标准化，企业就能做得越大。以人为核心供应链的产品是无法做大的，比如新东方的名师模式强依赖罗永浩、李笑来这些名师，就做不大，因为名师有限。相比之下，学而思走标准化的方式，反而能快速发展。蚂蚁前几年有在推一个叫「前端工业化」的概念，要又快又好又多地做项目，就得做标准化，依赖蚂蚁正式员工这种高级高价前端是做不大的，因为我们不可能无限制地招正式员工，同时现在在推的数字马力用工模式应该也是这个思路。另外，最近我有一个在想的问题是，Pro Code 如何做标准化，让门槛更低，让更多初级开发者，通过几个小时的培训就能上手 Pro Code 并完成项目研发。

10、市场即选择，产品即细分。进入一个烂市场，就像把一个短跑冠军扔进烂泥地，再厉害他也跑不快。如果你的目标是做大，那选前端这个市场，应该是不太可能实现的，池子太小，很容易就满了，而且大家又都那么卷。

11、心理学上有个名词叫作“孕妇效应”，简单来说，就是当你怀孕或者家里有人怀孕的时候，你会看到大街上到处都是孕妇。一个带来的思考是，做 Umi 框架时，要确认需求是来自用户的真实需求还是自己假想的需求，从用户需求看 Umi 框架的规划，避免孕妇效应。需求太重要了。用户需求是商业的起点，有了需求，才会有产品；有了产品，才会产生交易；有了交易，才会有商业；有了商业，才会有营销。

12、有个理论叫「焦糖布丁」理论。其核心思想是，顾客购买一件商品，并不是想拥有这件商品，而是想“雇用”这件商品帮他完成一个现实世界中的任务。要洞察到顾客想要完成的任务是什么，产品设计时如何能帮助顾客更好地完成这个任务。还是以 Umi 框架为例，用户想要一个框架其实不是要一个框架，而是要完成他们产品的研发任务。当你去思考用户任务时，开发产品的思路也就完全不一样了。康师傅最大的竞争对手是统一吗？可能不是，在一些场景下可能是外卖，因为他们都是为了完成一顿饭这个任务。

13、要保持竞争优势，不妨常问这几个问题。1）用户的任务有没有发生改变？是消失了，还是改变了？2）完成顾客的任务有没有更好的解决方案？3）顾客有没有什么新的任务？有没有你的机会？4）当你为顾客的任务提供了解决方案，顾客在实施时会遇到新的问题吗？你有没有准备好备用的解决方案？

14、商业上的事情没有什么标准答案，所有问题如果真有一个答案，那么这个答案就叫作“看情况”。情况不同，答案就不同。也叫「By Scene」，很多泛问题都是这样，没有答案的。比如数据流哪个比较好？By Scene，PC 和 H5 的选择就不一样。比如怎么做性能优化？By Scene。很多不给场景的问题，通常都难以得到好的答案。

15、人不识货钱识货，价格是商品的重要因素。有时价值决定价格，也有时价格决定价值。比如钻石就是价格决定价值，价格高就是钻石特性的一部分，茅台也类似。另外，肯定不是价格越低越好。比如送礼的场景，巧克力虽然好吃，但作为一些场景的礼物可能会因为价格太低，然后大家还不选择他。有些商品价格低，大家会觉得劣质，或者产生不信任感，商家提价之后销量反而会上去。

16、怎么对产品定价？3 个考量因素，成本、对顾客的价值、同行对竞对的价格。线上产品很少基于成本定价，比如星球、专栏、付费教程，都是基于对顾客的价值来定价的。苹果手机那么贵，也不是基于成本，而是基于价值定价的。

17、利润=（价格×销量）-成本。销量通常基础比较大，对比价格提升 5%、销量提升 5%、成本降低 5%，价格提升 5% 能带来的利润提升是最大的，所以商家通常会更愿意去做这件事。神奇的定价魔术！

18、有个概念叫消费者剩余。以星球为例，比如这个星球卖 299，而你觉得他值 1000，只要 1000 以内都愿意买，那多出来的 701 就是消费者剩余。相对的，也有商家剩余，假如对我来说算上时间等成本是 200，超过 200 我都愿意卖，那多出来的 99 就是商家剩余。所以，商家会希望最大程度地榨取消费者剩余，以顾客感知的最高价值成交，比如互联网杀熟。

19、一个汉堡的定价看似简单，但可能应用了价格歧视、差别定价、优惠券、限时促销（豪华午餐价格）、捆绑销售（“汉堡+可乐”套餐）、备选产品定价等方法。价格歧视、差别定价、优惠券都是为了满足不同支付能力和消费偏好的用户。比如用消费券的用户，并不是白白占了便宜，他们付出了时间成本。比如教育优惠就是一种价格歧视，好处是获得额外的客户同时培养未来的消费习惯，还有工业用电和民用电的价格差异、明星的代言费、购物中心给不同品牌的不同租金（海底捞可能免费）、咨询公司给大公司和小公司的不同报价等都属于价格歧视，让商家可以获得最大利润。

20、有个概念叫锚定价格。就是用户第一次看到或者听到的价格，会直接影响他之后对商品价格的判断，而他之后对价格判断的标准会接近第一次看到或听到的这个价格。这里贴个例子，「20世纪30年代，有两个兄弟在纽约经营着一家服装店。两兄弟一个是售货员，一个是裁缝。当一个顾客走进门来，售货员会观察顾客的喜好。当顾客询问价格的时候，他就会对着他那位裁缝兄弟喊：“这套西装多少钱？”“那套漂亮的西装吗？42美元。”裁缝兄弟会大声回复，以便让顾客清楚地听到这个价格。然后，售货员就开始了他的表演，他好像没听清楚裁缝说了什么，尽管大街对面咖啡店里的顾客都已经听到是42美元了，他还是会问：“多少钱？”“42美元！”这个价格又被重复了一遍。售货员好像听清楚了，他转向顾客说：“先生，这套西装售价是22美元。”这时候顾客不禁大喜过望，他会立刻从皮夹子里掏出22美元放在柜台上，带着西装迅速离开。」想到的几个场景应用是，1）卖课或产品时给个高的锚定价格，比如在介绍页说一句「有读者读完之后感叹说，这个星球价值 1 万」，2）有人问公众号广告报价时，你可能会让对方先报价，但其实可以自己先报个高点的比如 2 倍的价格，3）面试聊薪水也一样，先出价总是有优势的。

21、有个概念叫价格诱饵。比如你的商品有不同版本，A 59 块，A + B 125，这时你增加一个 B 125 的干扰项，会引导用户买更贵的 A + B 的版本。这在卖课等线上服务的价格表里经常会看到。低价版时给用户客户的选择，为了留住这部分用户。而增加一个与更贵产品价格十分近似但质量较差的产品版本，你就能够影响消费者比较的结果，他们会发现贵一点的产品突然显得更有吸引力了。
