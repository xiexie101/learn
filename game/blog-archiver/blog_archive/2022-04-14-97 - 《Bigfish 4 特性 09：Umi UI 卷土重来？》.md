# 97 - 《Bigfish 4 特性 09：Umi UI 卷土重来？》

> 4 月会更新 20 篇 Bigfish 4（含 Umi 4）的新特性，这是第九篇。

日常排查问题时，很多 Bigfish 框架的内部状态是看不到的，比如插件启用情况、appData 元数据、修改过的最终配置、修改过的最终 webpack 配置、修改过的最终路由、MFSU 的 module graph 信息等。

之前很多同学抱怨 Bigfish 黑盒，这也是原因之一。大家可能熟 webpack、熟 babel，但具体传了什么配置过去，在 Bigfish 框架中决定因素是很多的。除了用户配置，还有默认配置、插件逻辑、框架逻辑等。比如有些配置在某个插件里写死了，用户怎么配都是不生效的，这时就会比较沮丧。有些场景下，Bigfish 框架的维护者也都只能通过 patch node\_modules 来调试。

而如果有了这些内部状态，对于问题排查会更有帮助，同时大家也能更清楚 Bigfish 内部做了什么。

![](https://tva1.sinaimg.cn/large/e6c9d24ely1h18zekqga4j21ao0u0ada.jpg)

Bigfish 4 提供了 /\_\_umi 路由，dev 阶段可用，见上图。定位是开发辅助，大家也可以理解为是「丐版 Umi UI」。其提供的功能包括：

1、应用元数据  
2、配置  
3、日志  
4、路由  
5、MFSU 状态  
6、插件状态及耗时  
7、…

目前更多是开发调试方面的功能，之后会考虑把 Umi UI 中有价值的功能逐步加回来。
