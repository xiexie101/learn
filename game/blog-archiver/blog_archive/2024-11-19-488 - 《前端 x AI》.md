# 488 - 《前端 x AI》

> 整理了下 前端 x AI 的现状和 TNF x AI 可能会做的想法。同时这篇 WIP，想到了应该会持续更新，关注这部分的可以隔一段时间来刷一下。

1、前端 x AI 现状，大致了解了下，有这些场景和应用。

1）全自主 AI 软件工程师。比如 Cognition AI 公司的 Devin，在 SWE-bench 基准测试中，Devin 能够正确解决 13.86% 的问题。但 9 月发布后就再无消息。有点希望他别太强大，如果能取代 80% 的程序员就不好玩了。。

2）Code Copilot。VSCode 是 Github Copilot，Intellij 也有但没试过。另外就是风头正茂的 cursor 和 windsurf，前者正在用并已成为主力编辑器，后者据说风评不错但还没试。

3）代码生成，业界有名的应该是 v0.dev 和 bolt.new，蚂蚁内部也有 fecodex（改名 weavefox？） 可以做图生代码，基于图分析其可能用到的组件，然后用组件来拼。

4）其他。比如有 [https://midscenejs.com/](https://midscenejs.com/) 和 [https://www.meticulous.ai/](https://www.meticulous.ai/) 用于测试，比如蚂蚁内部用于 Mock 数据生成的，比如自动处理 commit 的 [https://github.com/tak-bro/aicommit2](https://github.com/tak-bro/aicommit2) 。还有啥？

2、tnf x ai。

我希望 tnf 能充分拥抱 AI，所以在这块会多考虑一些，一些是一些脑暴的点。

1）和 ai 编辑器结合，比如 cursor 可以生成配套的 .cursorrules 文件，让 cursor 更懂你的项目。

2）用 ai-digest 生成单文件的项目描述，交给 ai，然后让 ai 做修改建议。比如性能优化、最佳实践、潜在问题、代码改进等。性能优化可以结合 tnf，比如可以找出哪些组件没有用 loader 来加载数据，比如有没有配置好合理的 preload 策略等。

3）和 build 结合，把 stats 和 ai-digest 产物等交给 ai，让他提供构建优化建议。包括包尺寸优化、code splitting 建议、依赖优化建议等。

4）和错误结合，把出错信息、error code 文档、faq、相关 issue、答疑记录等交给 ai，让他分析错误并提供原因解释、解决方案建议、相关文档、示例代码等。

5）review 助手。把分支 diff 和 ai-digest 产物交给 ai，要求提供 review 建议，包括代码质量、安全问题、性能问题、可访问性建议等。

6）和配置结合，要求 ai 结合项目特点，给出合理的配置建议。

7）依赖管理助手。分析项目依赖，提供版本更新建议、安全风险、替代方案推荐、幽灵依赖检测等。

8）文档助手。

9）代码迁移助手。比如从 Umi 到 Tnf，把两个项目的区别告诉 AI，是否也可以用 AI 来做 Codemod。

10）eslint 快速修复工具。下午群里小虎提的问题，想了下，也可以尝试用 AI 来解，把 eslint 错误信息和 ai-digest 产物交给 ai，让他来改，然后人工 review，应该能提效好多倍。

11）答疑助手。把文档、faq、项目信息、报错信息、运行日志、相关 issue、历史答疑记录等喂给 ai，让他回答问题。同时可以参考 cursor docs，支持索引远程文档，比如 MDN、Ant Design、React 等，把多个文档的信息一起给 AI，可以提供更准确的回答。

12）交互式 AI 助手。提供交互式问答。

3、tnf generate with ai。

generate 如果只拿来 tnf generate page 就太浪费了，还应该提供 component、layout、hook、service、model、api、test、tailwindcss 等类型的生成，同时提供交互式命令和更多模版供选择，支持自定义模版等。

但如果仅仅是通过模版生成，就太弱了。generate 和 ai 结合应该能提供更强大的能力。先脑暴下可能的功能点。

1）提供自然语言生成能力。

```bash
tnf g --ai "Create a user profile page with avatar upload"
```

2）提供代码补全和修改建议。包括缺失的 import、推荐的 hooks、写了 TODO 的注释、类型定义、测试用例等。

```bash
tnf g suggest --ai src/page/index.tsx
```

3）提供重构建议。

```bash
tnf g refactor --ai src/components/UserList.tsx
```

4）AI 测试用例。

```bash
tnf g test --ai src/component/UserList.tsx
```

5）AI 样式。根据描述生成或更新样式。

```bash
tnf g styles --ai "description"
```

6）AI API。根据描述生成或更新 API 文档和 Schema，包含 API调用代码、类型定义、数据校验和错误处理。

```bash
tnf g api --ai path/to/swagger.json
```

7）AI Mock。根据 API 和 Mock 要求（JSON 或 Express 等）生成合适的 Mock 文件。

以及还有一些实现时的想法。

1）提供 accept 选项  
2）可能提供 gui 的交互模式，因为要提供 diff、apply 等能力  
3）抽象一层放 fishkit 里，方便后续提供给其他框架使用  
4）ai 相关依赖用 peerDependency，防止 tnf 内核太大
