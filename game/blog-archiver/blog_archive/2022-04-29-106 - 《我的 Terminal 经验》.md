# 106 - 《我的 Terminal 经验》

看完 [https://www.joshwcomeau.com/javascript/terminal-for-js-devs/](https://www.joshwcomeau.com/javascript/terminal-for-js-devs/) 入门 Terminal 后，可以继续学习一些进阶技巧。

软件别偷懒直接用原生的 Terminal，命令行是非常高频的操作，花时间选一个趁手的非常比好。我是用 iTerm2（[https://iterm2.com/），老牌软件，很稳。iTerm2](https://iterm2.com/%EF%BC%89%EF%BC%8C%E8%80%81%E7%89%8C%E8%BD%AF%E4%BB%B6%EF%BC%8C%E5%BE%88%E7%A8%B3%E3%80%82iTerm2) 一些高频快捷键：「⌘+←/→」切换 Tab，「⌘+D」横向新开 Pane，「⌘+⇧+D」竖向新开 Pane，「⌘+⌥+箭头」选择不同的 Pane。另外 [https://hyper.is/](https://hyper.is/) 和 [https://www.warp.dev/](https://www.warp.dev/) 也可以试试，Warp 看有同事在尝鲜，体验还不错。

shell 用 zsh，然后用 [https://ohmyz.sh/](https://ohmyz.sh/) 管理配置，配好了命令行体验能提升一大截。插件除了通用的之外，还额外开了 [https://github.com/zsh-users/zsh-autosuggestions](https://github.com/zsh-users/zsh-autosuggestions) 和 [https://github.com/zdharma/fast-syntax-highlighting。前者用于命令自动提示，后者用于命令语法高亮。](https://github.com/zdharma/fast-syntax-highlighting%E3%80%82%E5%89%8D%E8%80%85%E7%94%A8%E4%BA%8E%E5%91%BD%E4%BB%A4%E8%87%AA%E5%8A%A8%E6%8F%90%E7%A4%BA%EF%BC%8C%E5%90%8E%E8%80%85%E7%94%A8%E4%BA%8E%E5%91%BD%E4%BB%A4%E8%AF%AD%E6%B3%95%E9%AB%98%E4%BA%AE%E3%80%82)

以下是一些我非常高频操作的 alias 和辅助 function。

```bash
# 在 tmp 目录下新增随机临时目录，注意这个目录会在重启后被自动删除
alias cdtmp='cd `mktemp -d /tmp/sorrycc-XXXXXX`'
# 创建目录并进入
function mcd {
 mkdir $1 && cd $1;
}
# 进入 Finder 打开的目录
function cdf() {
 cd "$(pfd)"
}
# 修改 bash 配置
function bashrc() {
 vi $CURRENT_DIR/.bash_profile
 exec $SHELL -l
}
# git 相关
alias gcr="git checkout master && git fetch && git rebase"
alias gclean="git reset --hard && git clean -df"
alias gp="git push origin master"
alias gt="git status -sb"
alias ga="git add ."
alias gc="git commit -av"
# cd 相关
alias ..='cd ../'
alias ...='cd ../../'
alias ....='cd ../../../'
alias -- -="cd -"
# 快速打开 app
alias c='code .'
alias i='webstorm .'
alias e="stree"
```

三方命令我常用的有 autojump（目录跳转）、tig（git 日志）、bat（代替 cat）、ag（the\_silver\_searcher，代替 grep）、fd（代替 find）、dog（代替 dig）。更多命令，大家可以去 [https://jvns.ca/blog/2022/04/12/a-list-of-new-ish--command-line-tools/](https://jvns.ca/blog/2022/04/12/a-list-of-new-ish--command-line-tools/) 和 [https://github.com/ibraheemdev/modern-unix](https://github.com/ibraheemdev/modern-unix) 找找灵感。
