# 48 - 《信息处理工具的选择》

趁放假整理了下手上的信息处理工具。但是太多了，包括 Flomo、Notes、NotePlan、XMind、OmniOutliner、Logseq、Google Docs、Notion、Cubox、Eagle 等，不想想清楚什么场景用什么，会一团乱。

我梳理后的场景和工具见图。

![](https://img.alicdn.com/imgextra/i3/O1CN01H666fK1aYEB7IKxUp_!!6000000003341-0-tps-1200-859.jpg)

信息收集是纯收集，额外收益是可以防内容失效。收集区分图片还是页面，图片用 Eagle，页面用 Cubox。Eagle 的体验非常好，在页面中拖下图片就能入库 Eagle；Cubox 是新工具，还没具体上手用，看了评测和教程应该不错，这是苦于公司不让用 Evernote 的选择。

信息整理是把新信息整理到已有的知识地图中。这里区分结构化和非结构化的信息。

非结构化的我会用 Apple Notes 和 NotePlan。Apple Notes 优点是同步稳，我用于生活类的记录，比如旅行、游戏、理财、健康等；NotePlan 有点是可以加各种格式、图片等，我用于工作类的，比如周报、月报、绩效设定、团队工作等。

结构化的也有不同分类。如果有多维度，就需要用到表格，可以用 Notion 或 Google Docs Sheets；如果是无限层级的，可以用大纲或脑图的形式表达，大纲可以用 LogSeq 或 OmniOutliner，脑图可以用 XMind。

信息加工主要是写自己的理解。长的可以很长，短的一句话也算，核心是要理解后输出。这里区分长文和短文，长文用 Google Docs 和 iA Writer，短文用 Flomo。Google Docs 特点是功能全且稳定，写长文放心，可以更专注在内容上，比如正式演讲、小册、深度解析文章我会用这个；iA Writer 是 Markdown 编辑器，特点是简单好看，比如 Umi 发布、团队内的分享、MDH 周刊、RFC 提案我会用这个；Flomo 特点是简单和随手记录，比如日更短文我会用这个。
