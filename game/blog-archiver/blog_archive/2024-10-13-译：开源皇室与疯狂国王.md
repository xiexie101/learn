# 译：开源皇室与疯狂国王

> 原文：[https://world.hey.com/dhh/open-source-royalty-and-mad-kings-a8f79d16](https://world.hey.com/dhh/open-source-royalty-and-mad-kings-a8f79d16)  
> 作者：DHH  
> 译者：ChatGPT 4 Turbo

**编者注：作者表达了对开源领导者滥用权力的担忧。1）支持终身仁慈独裁者（BDFL）模式，开源不是民主，2）开源应保持中立，不要武器化。**

我坚定地支持 [终身仁慈独裁者（BDFL）模式](https://en.wikipedia.org/wiki/Benevolent_dictator_for_life) 的开源管理。这就是从 Linux 到 Python，从 Laravel 到 Ruby，以及是的，Rails 等项目保持其凝聚力、决策力和前进动力的方式。这是一个有着数十年成就的模型。但它不是天赐的权力。它不是绝对可靠的。

现在我甚至不愿意打开这个讨论。因为我已经经历过不少于[我的公平份额](https://world.hey.com/dhh/no-railsconf-faa7935e)对我自己的管理的恶意尝试，也见证了其他几个人的示众审判。一个项目内的争议性问题，或其外部社会的道德恐慌，很容易孕育出企图废黜的暴徒。他们将劫持然后颂扬\_社区的意志\_，就好像\_那个\_某种方式上理应得到天赐权力一样。

BDFL 模型的一半好处正是允许做出不受欢迎的决定，而无需委员会和官僚主义的迟钝风气！[开源不是民主](https://world.hey.com/dhh/open-source-is-neither-a-community-nor-a-democracy-606abdab)，无论我们是否接受，我们都从这个事实中受益。

因此，以下内容不是一个范畴性的论点。我相信开源皇室的社会效用。一个基于主动性、坚毅、贡献和技术卓越的美德加冕的皇室。

Matt Mullenweg 在 WordPress 的领域里赢得了他的王冠。他创建了系统，并且二十年来一直是它的主要倡导者和领导者。他的成就是显而易见的。半个该死的互联网都在运行 WordPress！这里有一个价值数十亿的行业，养活了主题设计师、插件制作者、托管公司和 Matt 自己的 Automattic 企业。这是一个一流的开源成功故事。

但这也是一个自从 [Automattic 与 WP Engine (WPE) 开战](https://world.hey.com/dhh/automattic-is-doing-open-source-dirty-b95cf128) 以来采取了黑暗转折的故事，理由是后者声称其收入的 8% 作为“提供更多回报”的什一税大约支付。提取的杠杆作用最初是一个伪造的商标主张，但自那以后升级为 [WPE 所称的敲诈行为](https://yro.slashdot.org/story/24/10/03/1354214/wp-engine-sues-wordpress-for-libel-extortion)，以及我所看到的似乎永无止境的戏剧性越界和违反开源规范的行为。特别是引入[登录忠诚誓言](https://wptavern.com/wordpress-org-login-gets-mandatory-affiliation-checkbox-following-wp-engine-dispute)，以及现在[对 WPE 的高级自定义字段（ACF）插件的征用](https://x.com/wp_acf/status/1845169499064107049)。

这个问题很复杂，所以我们来从最近的事件说起。我认为最不合理的升级行为是对 ACF 插件的征用。Automattic 首先回应 WPE 的诉讼，通过阻止后者的工程师访问 [WordPress.org](http://WordPress.org) 插件库来应对，该插件库用于分发更新和安全补丁。然后，它利用 WPE 不再能访问该注册表的事实，征用了包括评论和下载统计在内的插件！[ACF 条目现在指向 Automattic 自己的 Secure Custom Fields](https://wordpress.org/plugins/advanced-custom-fields/)。

对于一个始于“商标混淆”声明的争议来说，Automattic 现在将寻找 ACF 的用户引导到他们自己的插件上，这实在是讽刺至极。并且他们以 ACF 需要维护为由，提出这种前所未有的开源规范违规行为的理由，因为 WPE 被阻止了，无法提供维护，所以 Automattic 必须介入。我是说，这是什么情况？！

想象这种情况发生在 npm 上？想象 Meta 与 Microsoft（GitHub 的所有者，后者又拥有 npm）发生法律纠纷，然后 Microsoft 指示 GitHub 禁止所有 Meta 员工访问他们的仓库。然后 Microsoft 就接管官方的 React 仓库，将其指向他们自己的 Super React 分支。我们正在讨论的就是这种疯狂的事情。

武器化开源代码库是我们绝对不能允许形成先例的事情。它们必须保持中立领域。在不断的商业冲突中，它们是小瑞士。

这确实是我关注这整个糟糕事件的主要原因。如果这场争斗仅仅是两个价值数十亿美元的公司之间的争斗，如 Automattic 和 WPE，我就不会介意加入讨论。但是，此事件涉及的原则远远超出了他们两个。

在这种合同纠纷中使用像 WordPress 这样的开源项目作为筹码，并武器化其插件库，是对已经持续数十年的开源和平的危害，所有人都从这种和平中获得了红利。自 2000 年代初 [SCO-Linux 无理要求](https://en.wikipedia.org/wiki/SCO%E2%80%93Linux_disputes)以来，我们尚未面临过在开源领域如此潜在的恐惧、怀疑和不确定性的爆炸，这些是每个人都以为他们可以视为理所当然的基本事项。

所以虽然我一直试图避免牵扯个人情感，我会打破常规发出这个请求：Matt，不要变成一个疯狂的国王。我非常尊重你在 WordPress 及其它领域的工作。我也认识到感激情绪引发的问题，那些从我们的工作中获益更多而回馈较少的受益者所带来的诱惑。但这必须保持为道德批评，而不是商业征战。你不能仅凭一时兴起就强行索取你认为超出许可协议应得的东西。

请不要让我为像 Silver Lake 这样的私募股权运营商加油，Matt。不要让我希望他们提出紧急禁令来阻止对 ACF 的征用。

现在还不算太晚。是的，一些桥梁已经被烧毁，但将那些视为沉没成本。即使是孤立的，从现在开始继续这一征服所需的额外费用也不会值得。还有时间可以转变。达成一个适度的协议，让所有方都能有所保留。我恳求你去追求它。
