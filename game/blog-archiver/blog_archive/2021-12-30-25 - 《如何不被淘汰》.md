# 25 - 《如何不被淘汰》

> 针对 [https://t.zsxq.com/uV3Z3Fy](https://t.zsxq.com/uV3Z3Fy) 的回复。

程序员最不怕裁员了，不用担心，目前市场还是供不应求的。

名校、大厂、开源项目。前两者有加分，但不是全部。最近招实习生，我面的最出色的是非 985的，一路打 A 过的终面。开源项目也是加分项，很多人不做开源也很厉害。

具体什么重要，如果有仔细看[3 - 《我的面试套路》](https://sorrycc.com/my-interview-routine)，会找到一些答案。这里列了很多考查点，如果要挑最重要的，我会选「产出」，最好还是持续的。

什么是产出？能做开源、组件、框架、工具、体系化方案自然很好，但也有很多其他形式，日常的文档、文章、整理、总结、沉淀等，都算。我个人会在解完 bug 后会做记录，做完双 12 项目后会写写下次做如何更轻松再顺便提取组件和工具供下次使用，连续写了三个 PPT 和做完 SEE Conf 演讲评委后会整理演讲的 Checklist 和演讲套路，参加完其他同学的晋升会整理建议给到团队同学。

我开这个星球的目的，除了赚点生活费，也是为了更多地产出，一年之后回头看，如果有 365 篇短文，就有 365 次思考，同时对我的写作能力也是个提升。

回到你这，有些更具体的建议。

1、产出不一定在工作外的几个小时，和本职工作结合起来可能更轻松，并且工作绩效也会更好。可以找找工作中可以总结的点，以文档、组件、工具、分享的形式产出

2、leetcode 我很少刷，不能身体力行感知他的好处，对我个人而言，如果不去面外企、字节和微信，好像没具体的用处，学而不用则忘 🤦。如果要刷，也别仅仅是刷，要有产出，以文档或分享的形式，教是最好的学，会教才是真学会

3、最有效的学习是写轮子。Dan 也说：「Underrated way to learn something: try building a toy version of it.」「Instead of building your 101st shopping list, try building a toy version of React that can only render one button.」 。可以从 Promise、lodash 等简单的做起，熟练后再找一些当前火的项目，比如 Vite、Remix 等，轮子造地好，说不定还能出本册子搞搞副业

最后，如果产出不出来，那可能是输入不足，多些输入。等输入够了，就多产出，漂亮的履历是一步步「产出」出来的。
