# 176 - 《框架错误处理的五个级别》

如果你在使用框架或工具时遇到一个错误，你会怎么处理？Google？StackOverflow？非常有影响力的可以这么做。但在到达那个阶段之前，作为框架或工具的维护者，我们还可以做更多。

1、Level 1：不处理。让用户在 issue 里提问和搜索，如果之前有人遇到过解决过，就可能找到答案。问题是有很多干扰信息，用户可能需要翻好久才能找到正确答案，或者找不到。

2、Level 2：FAQ。把 issue 中的用户常见问题整理到 FAQ 文档，引导用户遇到问题先来 FAQ 里找。这里通常是经过梳理的问题介绍和方案，比较有保障。

3、Level 3：Error Code。FAQ 的进阶版，把用于遇到的问题按类型、来源做区分。更有辨识度，每个错误会有唯一 url 可访问到，可搜索，可分享。

4、Level 4：框架集成。把已经梳理的 FAQ 和 Error Code 规范化和工程化，集成到框架，框架遇到错误时自动匹配 FAQ 和 Error Code，让开发者通过错误码能自行解决大量问题，这对于 Bigfish 开发者和用户来说是双赢的。

5、Level 5：Doctor。框架集成的更进一步，通过静态分析的方式分析潜在错误，无需启动项目即可使用，适用场景更广，比如作为 CI/CD 环节的门禁。
