# 532 - 《make、n8n 和 dify》

这几天依次使用了 Make (原 Integromat)、n8n 和 Dify，说下感受。

1、这几个都是（基本）零代码的自动化平台，提供大量 SASS 服务的接入，也都支持 AI 类的 SASS 服务。能很快把多个 SASS 平台串到一起，自动化地做很多事。同时有提供 web hook、api、crontab trigger 等多种机制。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/FX7KnJ.png)

2、我先用的 [make](https://make.com/) 。make 是一个付费平台，免费用户 1 个月可以用 1000 个 Operation，其实也够用，不够的话可以多开几个账号。我写了个「每天早上提醒天气和当日 nba 比赛信息」的流程。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/UBDwBO.png)

然后每天会在 Telegram 上收到这样的消息。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/h59x8G.png)

3、然后用的 [n8n](https://n8n.io/)。n8n 是开源的，可以 self deploy 。1Panel 在应用商店搜「n8n」即可安装，绑定域名、注册证书，DONE。数据库默认用的 Sqlite，需要的话，定期备份 /opt/1panel/apps/n8n/n8n/data/database.sqlite 就好。

我写了个 WorkFlow，功能是「如果有新的豆瓣评分超过 6.9 的电影，Telegram 通知我」。1）先用 [r.jina.ai](http://r.jina.ai) 访问 [markdown 格式的豆瓣电影](https://r.jina.ai/https://movie.douban.com/cinema/nowplaying/hangzhou/)，2）再用 Gemini 提取内容并输出 json 格式的信息，3）然后用 [@telepilotco/n8n-nodes-kv-storage](https://github.com/telepilotco/n8n-nodes-kv-storage) 持久化最新的电影以避免重复推荐，4）最后发 Telegram 。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/uYKuKd.png)

然后，会在 Telegram 上搜到这样的通知。

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/tk1eWU.png)

另一个 Workflow 的功能是「当 [https://www.aiengineerpack.com/](https://www.aiengineerpack.com/) 上 Vol 4 时，Telegram 通知我」。

4、最后试了下 [dify](https://github.com/langgenius/dify)。dify 也是开源的，可以 self deploy，我用 docker 部署在本地。他相比前两个是更 AI 的，可以创建 Chat、Agent 和 Workflow。dify 包含不少模板，但我没有用他做啥具体的东西，因为感觉对于我的需求，可能更倾向于通过编程的方式用 \[\[799 - 《Mastra》\]\] 来做。（注：Mastra 有空会单独开一篇来写）

![](https://cdn.jsdelivr.net/gh/sorrycc-bot/images@main/uPic/kSOmOt.png)

5、一些感受。

1）平台的优势是 SASS 接入的量，缺点是不够灵活。一定你要做一些特殊的定制时，比如做跨 workflow 的数据存储，而平台又不支持这个功能，就比较费劲。可能前面只用了 5 分钟，而因为一个卡点可以卡上一天。

2）所以，如果你是程序员，我觉得还是用编码的方式更好，灵活性高，推荐 \[\[799 - 《Mastra》\]\]。
