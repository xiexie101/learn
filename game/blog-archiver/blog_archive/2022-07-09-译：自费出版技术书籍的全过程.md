# 译：自费出版技术书籍的全过程

> 原文：[https://ntietz.com/blog/start-to-finish-on-selfpublishing-a-technical-book/](https://ntietz.com/blog/start-to-finish-on-selfpublishing-a-technical-book/)  
> 作者：nicole  
> 译者：ChatGPT 4 Turbo

**编者注：作者（国外）自出版书籍的流程如下，成立公司、购买 ISBN 版号、找设计师设计封面、用 IngramSpark 印刷和发行、用 Quarto 把 Markdown 转成 PDF。**

我从 2015 年开始写这个博客，到了 2022 年，我的写作速度加快了。那一年我写了 37,000 字，比我之前写的 33,000 字还多。从那时起，它加速了。

有一刻，我意识到我在这里写作投入了\_很多\_时间和努力，并且想到将其打包成一本书的想法。这里的动机并不是为了赚钱，因为出版书籍很少能赚大钱。相反，有两个目的：给我一个我所投入工作的物理表现，以及给人们一种购买对我的写作表示赞赏的方式。这也是阅读过去作品的一种好方法；我已经购买了我喜欢的作者出版的[类似书籍](https://leanpub.com/computerthings2020/)，作为一种远离屏幕阅读他们作品的方式。

我的书现在可以印刷版购买了！它在[出版商网站](https://may11publishing.com/)上，也通过像 [Amazon](https://www.amazon.com/dp/B0CZB86NP1/) 这样的常见渠道出售。直接购买的选择对我们更有利，这也会以捐款的形式传递。

以下是我从开始到结束的做法[1](#logical-order)。顺便说一下：我从这本书中赚到的每一美元都将捐赠给支持跨性别权利。跨性别可见日快乐！

# 内容

第一步是获取内容，或者至少是一个目录。由于这是我博客文章的集合，大部分工作已经完成了。我真正需要做的只是决定包括什么。

书籍通常有一定的长度，这差不多就是我现在每年产出的量。这使得选择变得相当容易：2023 年将是它自己的一卷，而 2022 年及以前的年份也将是一卷，因为它们的总字数相同。

我只是将这些帖子复制到一个新目录中，因为它们已经全部是 Markdown 格式的了。

# 制作书稿

有了一处的内容，我需要将它们转换成我能打印的格式。我选择了 [IngramSpark](https://www.ingramspark.com/) 进行打印和分发，所以这意味着我需要内容的 PDF 版本。

我评估了一些不同的工具用于这部分，并最终选择了 [Quarto](https://quarto.org/)。我不太记得具体的理由了，但其他工具要么对我不友好，不愿意接受一堆分开的文件，要么格式我不喜欢且没有更改的选项。这是一个可靠的选择，我将在未来的卷册中使用它。至于我和朋友一起撰写的另一本书[2](#other-book)，我们可能会评估不同的工具，因为我们不需要将现有内容带入其中。

要配置好它，我更新了配置文件以指向所有我的内容。指向内容很直接，但我最终做了很多其他的自定义设置，这些我会一一讲解。完成的文件大致如下：

```yaml
project:
  type: book

book:
  title: "Technically a Blog Volume 0"
  author: "Nicole Tietz-Sokolskaya"
  chapters:
    - index.qmd
    - 2022-07-09-running-software-book-reading-group.qmd
    - 2022-09-11-going-to-recurse-center.qmd
    - 2022-09-24-rc-week-1-recap.qmd
    - ...

toc: true
toc-title: "目录"
toc-depth: 1

format:
  html:
    theme: cosmo
  titlepage-pdf:
    titlepage: formal
    titlepage-theme:
      elements: ["\\titleblock", "\\authorblock", "\\vfill", "\\footerblock"]
    titlepage-include-file:
      - includes/copyright.tex
      - includes/dedication.tex
    coverpage: none
    coverpage-title: ""
    coverpage-bg-image: "img/cover.jpg"
    links-as-notes: true
    include-in-header:
      text: |
        \usepackage{fvextra}
        \DefineVerbatimEnvironment{Highlighting}{Verbatim}{breaklines,commandchars=\\\{\}}
        \raggedbottom
        \usepackage{emptypage}
    documentclass: scrbook
    fontsize: "9pt"
    mainfont: "DejaVu Sans"
    from: markdown+emoji
    geometry:
      - paperwidth=5.5in
      - paperheight=8.5in
      - top=0.75in
      - bottom=0.75in
      - left=0.5in
      - right=0.5in

```

`project` 部分说明了这是什么类型的项目，因为不同类型的项目会使用不同的选项并产生不同的成果。然后 `book` 部分是我们输入一些元数据，比如标题和作者，我们列出了所有的章节，每个章节都有自己的 Markdown 文件。我确实不得不调整一些 markdown，因为我博客文章里通常使用的标题级别如 `# Heading` 会成为单独的章节，所以我所有的标题都不得不向内缩进一级变成 `## Heading`。在每个文件中调整这些真是太繁琐了！

下面这组陈述用于描述目录内容，并给出了一些简单的配置。我将深度限制为 1，这意味着只有章节标题，但能够轻松调整以同时包括目录中的小标题，这一点非常整洁。

`format` 部分是我花费最多时间和痛苦的地方。通常你会有一个 `pdf` 模块，但我有 `titlepage-pdf`，因为我使用了 [titlepage 扩展](https://nmfs-opensci.github.io/quarto_titlepages/)，它让我能够相对轻松地制作标题页。我选择的主题（`formal`）相当简约，但仍需要进行一些定制。回想起来，我或许可以通过使用 Quarto 为 PDF 提供的 [includes](https://quarto.org/docs/output-formats/pdf-basics.html#latex-includes)，_不使用_ 扩展来实现我想要的所有内容。

我这里有两个主要的 include，版权页和致谢页。这些是前言部分，在目录之前。它们通常也不编号！然后我在页眉添加了一些额外的 LaTeX，比如 `\raggedbottom` 避免执行垂直对齐[3](#justified) 和 `\usepackage{emptypages}` 用于移除空白页的页眉和页码。

这让我想起了：页码。

在这个项目之前，我没有太关注书中的页码位置或章节是如何开始的。结果是有一套标准！你在右侧页面开始新章节（传统上，对于英文书籍），这也是开始编号的地方。但你不对前言部分的页面进行编号，目录的页码是用罗马数字编号的。这些页码通常位于页面的外侧角落。我收到的第一个校样本中所有这些都偏了一个，以致页码在内侧角落，几乎没什么用。通过插入一个空白页修复了这个问题，但也花了我几周时间才收到第二个校样本！

表情符号也变成了一个问题，哦，我的写作中确实经常使用它们。我尝试寻找一个支持我使用的所有表情符号的字体，这就是我最终使用 DejaVu Sans 的原因。但第一个校样本中很多表情符号仍然缺失，我之前并没有注意到。我通过删除它们或用简单的字符表示来修复这个问题，因为此时更换字体很可能也会改变页码计数，进而改变封面模板，而且……这不值得麻烦。

那就是我制作内容 PDF 所做的所有事情。如果我早知道我现在知道的东西，过程会很快，但这是我第一次尝试，所以进度非常慢。

# 委托封面艺术设计

另一件我想要的事情是漂亮的封面艺术，如果经济上允许的话。我之前没有与插画师合作过，所以我不知道应该预期什么样的费用。幸运的是，一位复旦的同学 [Julia Evans](https://jvns.ca/) 给了我一个她之前合作过的插画师的推荐，并且他有时间接待一个新客户！

他合作起来非常轻松。我们通过几封电邮确定了项目的基本范围，并讨论了价格和时间线。然后他准备了一些概念草图来征询反馈。我给出了反馈，我们确定了一个我们俩都非常满意的设计！他非常灵活，合作起来很棒。项目结束时，他甚至建议如果预算允许，可以考虑双层封面——印刷成本略微增加，但内侧有抽象图案看起来会非常漂亮。

最终我们在我们讨论的时间线内完成了工作，预算也没有变化。书本本身比我预期的要晚完成，但这种推迟完全是因为我在后面的部分（下一节）做了一些晚期的决定，这导致了延期。

总而言之，以一个适度的费用，我得到了非常漂亮的封面艺术，它也被设计成一个我可以用于未来系列的模板。我迫不及待想再次与他合作另一个项目！

# 官僚主义和乏味

出版东西确实伴随着一些官僚主义。自行出版的书籍有时会由打印商提供 ISBN [4](#isbn)，无论是 IngramSpark 还是亚马逊。当你看到出版商列为亚马逊时，这就是原因。

我选择购买自己的 ISBN 并成为自己的印记，而不是使用打印商免费提供的 ISBN。这样可以提供最大的灵活性。如果你使用打印商的 ISBN，那么它只能在它们那里使用，并且每个分发它的其他平台可能会有不同的 ISBN，这样的话，诸如评论之类的东西可能会被拆分开来。ISBN 一旦分配就不能重新指定，所以一旦你决定如何做，就锁定了。如果你有自己的 ISBN，它可以在所有平台上保持一致。而且，拥有自己的印记看起来会更加专业。

这就引出了下一个话题。Imprint 是出版社用来出版作品的商标名称。一个出版公司可以拥有多个 Imprint。我听说成立一个有限责任公司 (LLC) 可以保护自己，于是我们（我和我的妻子）成立了 [May 11 Publishing LLC](https://may11publishing.com/)。宾夕法尼亚的 LLC 年费和报告要求很少，所以维护的成本不像特拉华的 LLC 那样昂贵。虽然为此设立 LLC 延误了项目进程，但这是值得的。我们还必须要开设一个商业银行账户，最终我们选择了 [Novo](https://www.novo.co/)，因为它的线上设置非常简便，无需电话通话。

ISBNs 正式属于我们的出版公司所有，而不是我们个人所有。这也为我们开启了出版其他作者的作品或者为其他人出版书籍的大门，如果有人对此感兴趣的话。我认为我们是一个非常棒的写作/编辑团队，而且我们正在与一位合作者探索另一个项目。

# 印刷和发行

我选择 IngramSpark 进行印刷的一个重要原因是他们也负责发行，因此大多数书商会自动采购这本书，当有人下单时，通过按需印刷来履行订单。而且他们提供大量的印刷选项，品质上佳，所以你可以得到你想要的确切产品。

我必须在早期就选择一个尺寸，以便插画师能够制作正确尺寸的封面，所以我选择了 5.5 英寸 x 8.5 英寸，这是一个相当标准的尺寸。其他细节：它是一个 [完美胶订平装本](https://en.wikipedia.org/wiki/Bookbinding#Thermally_activated_binding)，有哑光封面和黑白奶油色页面。选择页面尺寸的过程相当有趣。

在 IngramSpark 上创建书籍后，它会提供一个你可以下载的封面模板。这是一个 PDF 文件（我认为也有其他格式），你可以按照说明[5](#or-illustrator)把封面设计放进去。然后你可以上传你的内容文件和封面文件。他们会进行一些基本的验证，然后你就可以订购校对样本了！

我订了一个加急服务，以便快速上传迭代文件。你有 60 天的时间上传修订后的证明文件，在那之后，新修订将收取费用。费用虽少，但比我这次收费的还要多，所以如果可能的话，我想避免这笔额外开支。第二份校对副本我没有选择加急，因为我对它将是最终版本有足够的信心。果然如此！

它到达后，[我的内部编辑](https://sokolskayatranslations.com/) 和我进行了抽查，一切看起来不错，所以我启用了发行。这意味着它进入了 IngramSpark 发布书籍的数据库，书商开始采购！不到一天的时间就在亚马逊上通过第三方卖家上架了，大约一两天亚马逊和 B&N 正式列出。看到 _我的书_ 在亚马逊上出售对我来说还是相当不真实的！

顺便一提：如果你确实在亚马逊上买它，点击进入“其他新书”选项，并购买由 [Amazon.com](http://Amazon.com) 直接卖出并发货的那个，而不是主要列表上的卖家。有些第三方会列出一堆按需打印的书，并声称有现货，而亚马逊则诚实地说它缺货。如果你订购了一本，他们会订购一份来发送给你；但对于另一个卖家也是如此！而且直接从亚马逊购买更便宜。

或者，直接[从我们这里购买](https://may11publishing.com/)，我们会捐出我们得到的更多收益。这就引出了钱的问题。

# 资金方面

以下是我们在这个项目上花费的所有费用和成本：

*   **封面艺术：** $200，可在未来的版本中进行小修改后重复使用，所以未来的成本会更低
*   **国际标准书号（ISBN）：** $295 买 10 个。我这里用了一个，所以这个项目的成本是 $29.50，剩下的可以用于未来的项目。如果你只买一个 ISBN，要 $125，所以如果长远来看你至少需要三个，那么批量购买会更便宜。它们是不会过期的。
*   **校对副本：** 第一份因为加急服务所以是 $17.45，第二份则是 $9.35。
*   **宾夕法尼亚州有限责任公司（PA LLC）设立：** $125，从 2025 年起我们每年都需要提交年报，每份年报的费用为 $7。

这些都是项目的成本，总共是 $521.80。不过如果你看平摊成本的话，实际上更像是 $121.30（假设有限责任公司和封面艺术适用于五个版本，并且所有的 ISBN 都被使用了；这个计算不是完美的）。如果你可以接受不拥有 ISBN，并且自己制作艺术品的话，成本可以降低到只有校对副本的费用！

那么我如何从这个项目中赚钱呢？当然是通过销售。你可以在 IngramSpark 设置价格，然后有两种截然不同的购买方式和补偿。

*   如果他们直接从 IngramSpark 购买（你可以设置一个电商页面），价格为 $20 加上运费（大约 $24 总计）。如果他们这样购买，有 $3.50 的额外打印费用，而打印成本为 $5.33，所以我能得到 $11.17。
*   如果他们通过 Amazon 或其他书商以 $20 的价格购买（包括运费或不包括，取决于卖家），这将产生 40% 的批发折扣（我能设置的最低折扣），我能得到 $6.77。

是的，如果你直接从 IngramSpark 购买，我能获得的销售收入几乎是两倍。而且这是在设置较高价格和允许的最低批发折扣，以最大化收入的情况下实现的。

再次申明，从这个项目中获得的 _任何_ 利润都将被捐赠，用于支持跨性别权利。我们不会用这些收入来偿还费用：$521.80 是我们贡献的一部分。所以如果你从我们的网站购买一本书，这意味着你**得到一本书，我感到内心温暖，并且你的钱有助于保护跨性别权利**。

# 再做一次

我愿意，并且将会再做一次！我打算在休息一段时间后的 2023 年夏天或秋天准备 2023 年的卷。然后，我计划每年或每隔一年做一次，因为我现在在博客上写的东西比以往任何时候都多。

这一次我打算做一些不同的事情。值得注意的是，我没有所有的初始成本，而且我相当自信第一个校对副本将是唯一的校对副本。除此之外，我打算简单地去除所有无法渲染的字符，然后就完成了。
