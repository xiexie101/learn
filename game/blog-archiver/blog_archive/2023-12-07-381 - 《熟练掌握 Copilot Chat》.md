# 381 - 《熟练掌握 Copilot Chat》

今天翻到 [https://www.youtube.com/watch?v=SZVCJRUADc4](https://www.youtube.com/watch?v=SZVCJRUADc4) 这个视频，有些收获。然后顺着文档多学了下 Copilot 的使用，做下记录。作为一个每天都在用的工具，摸摸透还是有好处的。

1、唤起 Copilot Chat 有多种方式，除了侧边栏的 Chat Tab，还有两个快捷键，分别是「⌘+I」和「⌘+⇧+I」。前者唤起一个 inline 的对话框，后者是唤起一个全局的对话框。个人用下来感觉后者更好用，比如前者我在选中一段代码之后让他加 console.log 信息，他会给整个文件都加上，后者则是正常的。

![](https://img.alicdn.com/imgextra/i4/O1CN01ktyGsb1yMp4w5KfHG_!!6000000006565-2-tps-1694-1666.png)

2、输入 /help 可以看到整体的帮助信息，见上述截图。可以看到，1）有 3 个 @ 开头的 Agent，分别是问项目代码的 @workspace、问 VSCode 的 @vscode 和问 Terminal 信息的 @terminal，2）以及常用的命令，包括解释代码用的 /explain、帮你写测试的 /tests、修复编辑代码的 /fix 和新建 workspace 的 /new。

3、举几个例子。

比如我要给 function foo(num) { return num + 1 } 加日志信息，可以按「⌘+⇧+I」输入「add console.log info」，等代码出来后点右上的「Insert at Cursor」按钮，他会替换我之前选中的代码。

比如我要给 function foo(num) { return num + 1 } 加测试用例，可以按「⌘+⇧+I」输入「/tests generate test using test() function」，等代码出来后点右上的「Insert into New File」按钮，他会创建一个文件，然后保存到需要的位置就好。

比如我想搜索所有处 dist 目录之外的 js 文件，可以按「⌘+⇧+I」输入「@terminal search all js files exclude dist dir」，等代码出来后点右上的「Insert into Terminal」，就可以在 Terminal 里执行了。

比如要把 TypeScript 转成 JavaScript 代码、等等。

4、在 Chat Tab 里交互时，可以输入 `#file:` 然后可以选择文件，然后就可以基于文件进行提问了。

5、如果你在 Terminal 执行命令失败，左边会出现一个魔法按钮，点击会弹出 Modal 和你解释原因，并给出建议。

![](https://img.alicdn.com/imgextra/i4/O1CN01SfoEeY1JhWjdG9yTj_!!6000000001060-2-tps-1144-466.png)

6、提 Commit 和提 PR 时也会有一个魔法按钮，可基于改动自动生成。

7、有一个叫 VS Code Speech 的官方插件， [https://marketplace.visualstudio.com/items?itemName=ms-vscode.vscode-speech](https://marketplace.visualstudio.com/items?itemName=ms-vscode.vscode-speech) ，安装后会在 Chat 的输入框右边出现一个语音按钮，点击后就可以用嘴编程了。
