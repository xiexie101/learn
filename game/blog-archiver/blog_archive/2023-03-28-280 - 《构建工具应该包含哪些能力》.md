# 280 - 《构建工具应该包含哪些能力》

> 这篇文章回答的问题是，如果要实现一个构建工具，应该覆盖哪些能力？本文基于 [tooling.report](https://bundlers.tooling.report/) 的内容做了一些调整。

> 注：感觉必备的基础能力我加粗了。

源码转义。

*   **支持 TypeScript**
*   **支持 JavaScript 高级语法**
*   **压缩**（包括模块、语句合并）
*   **Tree Shaking**
*   **常量标志**（比如 process.env.NODE\_ENV 的处理）
*   图片压缩（大家都基于 [imagemin](https://github.com/imagemin/imagemin)）
*   SVG 压缩（大家都基于 svgo）

输出格式。

*   **CommonJS**
*   ES Module

非 JavaScript 资源。

*   **CSS**（包括 CSS Modules、Inline 引入、CSS Extract、以 URL 的方式引入）
*   **HTML**（包括 Inline 载入脚本、外链载入脚本、Preload 图片和字体等、Preload 依赖脚本）
*   **依赖**（包括产出 CSS 里依赖的三方文件、相同资源去重）
*   入口（包括支持非 JS 的入口文件）
*   **图片**（包括 data-url、url 两种形式）
*   Service Worker
*   Binary（包括 array buffer、url 两种形式）
*   自定义类型

引入模块。

*   **Common JS**
*   **ES Module**
*   来自 Node Modules 的依赖

Hashing。

*   略

Code Splitting。

*   **Dynamic Import**（即 `import()` 语法）
*   单页面多入口文件（共用 runtimeChunk）
*   多页面多入口文件
*   Chunk 间共享
*   Worker 和非 Worker 之间共享
*   Worker 和 Worker 之间共享

个人补充一些如下。

*   **Resolving**（包括 alias、externals、package.json exports 等）
*   Source Map（怎么调试）
*   **publicPath**（怎么加载资源）
*   Targets（包括 Modern 产物、Legacy 模式等）
*   补丁方案
