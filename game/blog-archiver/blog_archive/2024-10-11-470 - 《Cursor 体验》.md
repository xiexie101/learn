# 470 - 《Cursor 体验》

1、功能介绍。

1）Tab。和 Github Copilot 类似的功能，编写的时候做提示，只要一路 Tab 下去就好。  
2）Chat。和 ChatGPT 类似的功能，可以输入文字进行交互。支持 `@Codebase` 问代码库，`@Web` 问搜索网页，`@Docs` 问文档，以及 `@` 文件和目录等。快捷键是 ⌘L，⌘⇧L 可以新增要提问的代码块。  
3）⌘K。可以选中代码块，然后 ⌘K 进行编辑，也可以不选中；还可以在 Terminal 中输入 ⌘K 用自然语言写命令（Github Copilot 也有）。  
4）两个 Beta 功能，在 Cursor Settings 中配置开启。1 是长语境，可以 @ 整个目录进行聊天，2 是 Review，可以针对 Diff 进行 Review。  
5）Cursor Composer。

2、Cursor Composer。

感觉 composer 才是他的杀手级功能，可以做多文件编辑，想象空间就比较大了。比如构建脚手架、跨文件实现复杂功能、重构代码、提取函数到其他文件、生成测试用例等。通过 ⌘+I 调出，⌘+⇧+I 全屏调出。

示例一，参考 [https://www.youtube.com/watch?v=W4QmPwNwt8E](https://www.youtube.com/watch?v=W4QmPwNwt8E) ，快速搭建一个脚手架。

1）用 [Claude AI](https://claude.ai/) 或 ChatGPT 生成需求，Prompt 比如 `come up with step by step instructions for an ai to create a new react app using vite, tailwindcss, typescript, and shadcn/ui. this app is a simple todo list app`，然后保存为 plan.txt 文件。（当然，你手写需求也可以）  
2）⌘+⇧+I 调出 Cursor Composer，输入 `use the instructions in @plan.txt to create the app`。

示例二，做代码重构，比如提取组件到其他文件，参考 [https://x.com/asidorenko\_/status/1843041647313174619](https://x.com/asidorenko_/status/1843041647313174619) 。PROMPT 比如 `extract shared ui components into /components`。

示例三，10 分钟创建一个 iOS App，参考 [https://x.com/ammaar/status/1834348042637521031](https://x.com/ammaar/status/1834348042637521031) 。

示例四，创建 Mac App，参考 [https://x.com/ammaar/status/1828129847014490519](https://x.com/ammaar/status/1828129847014490519) 。

3、bolt.new。最近新出了一个 bolt.new ，个人尝试了下，感觉 Cursor Composer 就能覆盖其大部分功能，而且大多人也没有在线编码的习惯，所以对我来说没啥尝试的必要。

4、价格。

1）Cursor 有免费版，这相比 Github Copilot 是优势，让更多人可以体验。  
2）Pro 版每月 20 刀，有点小贵，年付则稍微便宜一些。但新注册会员有两周的试用期，所以如果你不想花钱，可以每两周换个新邮箱（支持用 gmail 的别名）

5、体验。

> TODO，一到两周后慢慢补充。

1）可以用来读源码，选中文件（可以选多个），然后问「请帮我理解这个文件的代码实现。」

![](https://tcc.sorrycc.com/p/4b8543b6-5142-4628-be25-f477728439a9.png)

2）他会猜你下一步做什么，并且把光标放到合适的位置，这一点比 Github Copilot 要好。但很难描述场景，只可意会，让你体会「一路 Tab」的快感。
