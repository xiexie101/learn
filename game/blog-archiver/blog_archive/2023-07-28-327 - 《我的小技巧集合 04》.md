# 327 - 《我的小技巧集合 04》

> 每凑够 10 条发一篇。

*   目标是「一定要做到的事」，而不是「想做的事」。
*   动手做事前，先问一句「为啥是我来做」，如果能找到合适的人代替你做，那就可以省去大段时间了。不要觉得这是把不重要的事情分配给别人而有内疚感，相同任务对于不同人的重要程度也是不同的。
*   Mac 微信搜索栏里输入「:recover」调出「微信修复工具」，选上「使用默认浏览器打开链接」，可以解链接默认使用微信浏览器打开的问题，[图](https://img.alicdn.com/imgextra/i1/O1CN01mvbf5E1DJoT7SQFOQ_!!6000000000196-2-tps-1206-1682.png)，发现 3.8.1 这个版本已经失效。目前我的办法是拖住链接，切换到 Google Chrome，再放开来实现用浏览器打开。
*   一个小知识，nodejs 项目，项目里安装了 events 包，你应该怎么 require 到安装的 events 包而不是 node 内置的 event 包？答案是require(‘events/’)，多一个 / 的后缀。
*   pnpm run /^test:/ 可以让 pnpm script 并行执行，[来源](https://twitter.com/sitnikcode/status/1678365174363766784)
*   一个开源库，如果作者没有留邮箱，怎么联系上他呢？可以试试从 git commit 信息里找找看，通常开发者都会通过 git config 配置上
*   VSCode 有两个容易被忽略的快捷键，1）Ctrl+R，可以快速打开或切换项目，2）Ctrl+Q，可以切换 Panel 。
*   新发现一个 Chrome 插件 [Markdownload](https://chrome.google.com/webstore/detail/markdownload-markdown-web/pcmpcfapbekmbjjkdalcgopdkipoggdi)，可以以 Markdown 的格式保存当前网页的内容。
*   我现在经常需要同时开两台 Mac，Apple 自带的「Universal Control」可以让一台机器上的键鼠控制另一台机器，但有个问题是经常不灵。不灵的时候，有几个可以尝试的方法，1）两台电脑分别尝试下「移动鼠标到屏幕边缘」，2）合上一台电脑后重新打开。
*   处理 Side Effects 和 Tree Shaking 时，为啥 foo.bar; 有副作用而 window.bar; 没有副作用？因为 foo.bar 可能是 getter 或者 foo 可能是 proxy 。感兴趣的可以把相关代码贴到 Rollup 的 Repl 中看产出结果。
