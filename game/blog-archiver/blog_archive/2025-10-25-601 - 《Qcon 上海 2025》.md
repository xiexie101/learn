# 601 - 《Qcon 上海 2025》

> 记录下流水账，参展、议题、AIGC、人和上海。

1、参展。

之前都是以观众、讲师、嘉宾和出品人参会，这次第一次作为参展商参加 QCon，以「蚂蚁开源」的名义。

会议前几天才收到的通知，比较匆忙，也没啥经验，啥都没准备就去了。另外来的几位同事也是没啥经验。其他展台都有准备活动、礼品、大屏、抽奖甚至充气人偶啥的，我们就干聊，哈哈。所以人流就相对要少一些。不过我们也没有引流的需求，都是开源产品，也不能引 star 啥的。

人流第一天多，后面两天就少了不少，因为是同一批人，该参加的都参加过了，后面两天大多专心参会了。

蚂蚁开源一共放了 3 个产品，Neovate Code、Galacean Effects 和 一个 Markdown 渲染引擎。Neovate Code 被问的最多，因为和 AI 最相关吧。被问的最多的问题是「你们相比其他的 Code Agent 有啥优势」，被问了 N 遍，其他同事都会回答了…

参展了好多 Code Agent 的，有看到 iflow、kiro、trea、vecli 等。kiro 摆了 3 台电脑，但没让体验，只引导注册… 目的性好强。火山引擎拍照产出 minecraft 照片同事打印给相框还挺有意思的。其他大多是关注或加群送盲盒之类的。

2、分享。

参展商的胸牌是可以去听分享的，所以第二天展台人流少了之后就去听了不少 Code Agent 领域的分享。有 trea、vecli、qoder、vibe coding 等，trea 听了两个分享，但相似度估计有 60% - 70%… 基础内容都是他们的架构演进 + 产品介绍。

Code Agent 领域感觉没啥秘密，大家做的和要做的项其实都差不多，所以聊的内容也没有差太多。基本上就是这些内容，vibe coding、spec driven、多 agent、agentic loop、content engineer、sub agent、humain in the loop 等。

感觉每家公司都有 cli code agent。听分享才知道火山引擎也有 cli 形式的 code agent，叫 veCLI，核心特性应该是他们自己搞的一套 zOne 的多 agent 的引擎。trea 有人问他们是否有 cli code agent 的计划，没有正面回答。1022，qoder 发布 cli；1023；kilo coder 发布 cli；1025，kimi 发布 [kimi cli](https://github.com/MoonshotAI/kimi-cli)。

另外也听了一些 aigc 的分享，爱奇艺和 bilibili 等公司的分享，对如何生成沙雕剧有了个大概的认知。

3、人。

见了一些老朋友。也和不少同学交流了 ai 领域的经验。感觉大家都在搞 ai，有些是通用的，但更多是垂类的。
