# 272 - 《新知识学习法和 Rust》

> 我也刚学没多久，结合最近看的一门书，班门弄斧下，哈哈。有不对的，欢迎指正。同时我觉得这个方法也适用于学习 Rust 之外的任何新知识。

1、关于如何学新知识有个简单的公式，「掌握最小知识集 + 用 + 持续改进」。1）第一步是掌握最小知识集，很多知识的最小知识集其实很小，比如设计的是简洁 + 留白，比如 5 年前我还为 dva 梳理过一个最小知识集 [dva-knowledgemap](https://github.com/dvajs/dva-knowledgemap)，2）第二步是用，刚开始时肯定是用不好的，有些人会得出「也需我不适合干这事吧」的结论，也有些执行力强的人会坚持下来，3）第三步是改进，不断完善前一步的「用」法。

2、此外还有个 Tip 是「专注」。这会让你尽快度过那个「用」不好或者说优点笨拙的阶段。因为专注的时候，时间会飞速流逝。

3、我们试着把这个公式套在 Rust 学习上。Rust 的最小知识集是啥？我觉得是环境和编辑器配置、[Cargo](https://doc.rust-lang.org/cargo/)（工程化）、基本和复合的数据类型、包和模块、所有权和借用、模式匹配、泛型和特征、错误处理、生命周期、[NAPI-RS](https://napi.rs/)（Node 生态必备）。我个人是看的 [Rust 语言圣经](https://course.rs/)，比较推荐，作者不仅文档写地好，主要是还有点风趣。

4、然后就马上进入「用」的阶段。具体怎么用 Rust？我个人的方法是看源码 + 模仿。魔方就是写个对应库的 toy 版本，比如看完 Farm 可以写个 toy-farm，看完 Dum 可以写个 toy-dum。刚开始「用」肯定会各种不顺，别担心，这是正常的，大家都一样。能坚持下来的就学会了，不能坚持的可能需要过个半年再学一轮。当然，如果能和你的工作结合起来，就更好了，不仅双赢（个人技术 + KPI），还能带薪学习。蚂蚁近期应该会有不少同学能投时间做 Rust，我个人计划是 40%，感谢老板和公司！

6、然后到「改进」这一步，我的方法是找熟悉 Rust 的同学给我写的 toy version 提建议。另外我想到的就是开源，然后如果你写的工具对别人有用，没有石沉大海，也可能会有同学给你提 PR 和建议。

7、Rust 学习曲线陡？我觉得之前可能是，现在有了 Github Copilot 和 ChatGPT，个人感觉已经降了非常多。我在「用」和「改进」的阶段会大量借助这两个工具。比如，1）翻源码遇到不会的就把代码 copy 出来，加 prompt「请问这段代码啥意思？」，除非是问一些新出的库，否则 ChatGPT 给的答案基本上很准，2）写代码时忘记怎么写？写个注释，然后等 Github Copilot 给提示。

这基本上是我目前学 Rust 的方法，希望对你有用。
