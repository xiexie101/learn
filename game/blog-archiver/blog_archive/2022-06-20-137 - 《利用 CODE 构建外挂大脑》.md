# 137 - 《利用 CODE 构建外挂大脑》

这个 CODE 不是代码，而是来自《Build a Second Brain》中介绍的方法。我翻了下前 7 章，主要就是讲 CODE，翻完还挺有收获，尤其是 Distill 里的渐进式总结法和 Express 里中级包的概念。

CODE 即 Capture（捕捉）、Origanize（整理）、Distill（提炼） 和 Express（表达） 四步法，指引我们每天如何面对无尽的信息流。

Capture（捕捉）指把有用的信息保存下来。为什么保存？任天堂退出屏幕有一条信息「Everything not saved will be lost.」。保存什么？不要全篇保存，不要长篇累牍地保存，不要保存大片大片的笔记！一句话总结是「保留有共鸣的东西」，换我的话说是「保留让你茅塞顿开的金句」。如果不好判断，可以问一些问题，比如是否激励了我、是否有用、是否让你发出「wocao」的惊叹。

捕捉的信息多了后，就需要 Organize（整理），否则会一团乱麻。需要杜威十进制系统吗？不用！本书作者发明了 PARA 系统命名法，即 Project（短期在做的项目）、Area（长期关注的领域）、Resource（未来可能参考的资源）、Archive（已完成或搁置的存档）。这在社区已经有大量应用，我个人将其用于笔记、书签、任务管理等领域，但用的是改良过的 IPARA，即在前面加一个 I（Inbox）作为收集箱。

做笔记有个悖论是：收集的笔记越多，信息量就越大，回顾这些信息所需的时间和精力就越多，而他们能做的时间就越少。Distill（提炼）可以解这个问题，提炼是从几百页原材料的复杂见解中提取「能传达核心信息的一两句话」，比如爱因斯坦用 E=mc2 总结了他的新物理学理论，比如毕加索的公牛还原了公牛的本质。提炼是为未来的自己做的。作者有一套「渐进式总结法」可提升笔记的「可发现性」，分捕获笔记（大段文字）、加粗（要点）、高亮（特别有趣或有价值）、摘要（自己的话概括）四步，感觉像可放大缩小的地图。

Express（表达）是将消费转向创造，自己有成就感，还能对他人产生积极影响。为什么创造？前三个步骤都是为了一个最终目的：与他人分享你的想法、故事和知识。如果知识不能帮助任何人或产生任何东西，将毫无意义。怎么创造？作者给的方法是「中级包」，利用小思维。报告、交付物、计划、文字、图形、幻灯片都是中级包，套用到程序员，算法、模式、原则、模块、框架、依赖包等都是中级包。完全依靠灵感的作家或艺术家，这种神话只是神话。不要在一张白纸开始你的项目，而是用一组积木（中级包）开始。你拥有的积木越多，就越容易建造出有趣的东西。
