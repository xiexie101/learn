# 设计方案：扑克花色主题视觉升级
**日期**：2026-01-15
**状态**：已确认

## 1. 核心目标
为“转盘”和“任务弹窗”增加视觉记忆点，通过**扑克牌花色**强化四个任务类别的区分度，增强游戏的仪式感。

## 2. 视觉映射方案
将四个任务类别与扑克牌花色进行符号化绑定：

| 类别 (Category) | 颜色 (Color) | 花色 (Suit) | 象征意义 |
| :--- | :--- | :--- | :--- |
| **异化 (Detournement)** | <span style="color:#B22222">🔴 红色 (Firebrick)</span> | **♥ 红桃 (Hearts)** | 情感、内心、打破常规的热情 |
| **共谋 (Conspiracy)** | <span style="color:#228B22">🟢 绿色 (Forest Green)</span> | **♣ 梅花 (Clubs)** | 生长、交织、自然、协作 |
| **祈梦 (Dream)** | <span style="color:#800080">🟣 紫色 (Purple)</span> | **♦ 方块 (Diamonds)** | 珍贵、星辰、梦境碎片、神秘 |
| **共振 (Attunement)** | <span style="color:#0047AB">🔵 蓝色 (Cobalt Blue)</span> | **♠ 黑桃 (Spades)** | 智慧、感知、理性、频率 |

## 3. 界面变更点

### 3.1 转盘 (The Wheel)
- **扇区图标**：在每个扇区的文字标签上方或背景中，增加对应的花色图标。
- **样式**：图标使用半透明水印效果，或者与文字排版结合，保持简洁。

### 3.2 任务卡片 (Task Card)
- **背景纹理**：在任务卡片的右上角或背景中，增加巨大的、低透明度的对应花色作为装饰水印。
- **标题装饰**：任务类别（CATEGORY）旁边可以加上小图标。
- **UI 反馈**：抽中任务时，卡片的主色调应严格跟随该花色的颜色。

## 4. 技术实现建议
- 使用 **SVG** 绘制四个花色，确保在不同屏幕下的清晰度。
- 将 SVG 代码直接内嵌或作为 Sprite 引入，减少请求。
- CSS 变量控制颜色，方便统一调整。
